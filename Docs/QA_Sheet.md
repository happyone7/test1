## 1. QA 프로세스 가이드 (QA 실무 인원용)

### 1.1 테스트 환경 준비

1. Unity 에디터에서 프로젝트를 엽니다 (`/mnt/c/UnityProjects/test1`)
2. 씬 파일 확인: `Assets/Scenes/Main.unity` (또는 프로젝트의 메인 씬)
3. 에디터 상단 메뉴에서 `File > Build Settings`를 열어 씬이 빌드 목록에 포함되어 있는지 확인합니다
4. Console 창 (`Window > General > Console`)을 열어 에러/경고를 실시간 모니터링합니다

### 1.2 플레이 모드 진입 방법

1. 에디터 상단의 **Play 버튼** (삼각형)을 클릭합니다
2. 게임은 **타이틀 화면**부터 시작됩니다
3. 화면 전환 흐름: `타이틀 -> 허브 -> 인게임 -> 런 종료 -> 허브`

### 1.3 화면별 진입 경로

| 화면 | 진입 방법 |
|------|-----------|
| 타이틀 | Play 모드 진입 시 자동 표시 |
| 허브 | 타이틀에서 "시작" 버튼 클릭 |
| 인게임 | 허브에서 스테이지 선택 후 "출격" 버튼 클릭 |
| 런 종료 | 인게임에서 웨이브 클리어 또는 기지 HP 0 도달 |
| 허브 복귀 | 런 종료 패널에서 "Hub" 버튼 클릭 |

### 1.4 PASS/FAIL 판정 기준

- **PASS**: 기대 결과와 실제 결과가 일치하며, Console에 Error 로그가 없음
- **FAIL**: 기대 결과와 실제 결과가 불일치하거나, Console에 Error/Exception 발생
- **WARN**: 기능은 동작하나 Warning 로그가 과도하거나 시각적 결함이 있음
- **SKIP**: 전제 조건 미충족으로 테스트 불가 (사유 기록 필수)

### 1.5 버그 발견 시 기록 방법

1. **상태** 컬럼을 `FAIL`로 변경
2. **결과 메모** 컬럼에 다음 정보 기록:
   - 증상 설명 (1~2줄)
   - Console 에러 메시지 (있을 경우)
   - 재현 빈도 (항상/간헐적/1회)
3. **스크린샷**: `Assets/Screenshots/qa_[항목번호]_[날짜].png` 경로로 저장
   - 에디터에서 `Game` 뷰 활성화 후 `Windows > Screenshot` 또는 OS 캡처 도구 사용
4. 심각도 태그 추가:
   - `[P0]` 게임 진행 불가 (크래시, 무한 루프, 화면 미표시)
   - `[P1]` 기능 오작동 (수치 오류, 버튼 미반응)
   - `[P2]` 시각/UX 결함 (레이아웃 깨짐, 색상 불일치)
   - `[P3]` 개선 사항 (폴리싱)

## 3. 상세 테스트 가이드

### 3.1 타이틀 -> 허브 전환 테스트 (T-03, H-01, H-02)

1. Play 모드 진입
2. 타이틀 화면 확인: 배경, 로고, 3개 버튼 정상 표시
3. "시작" 버튼 클릭
4. **확인 사항**:
   - 타이틀 화면이 완전히 사라짐 (Canvas 비활성)
   - 허브 화면이 표시됨
   - 상단 바에 Bit/Core 재화 표시
   - 하단 바에 스테이지 드롭다운/출격 버튼 표시
   - Console에 Error 없음

### 3.2 스킬 구매 테스트 (SK-07 ~ SK-14)

1. 허브 화면 진입
2. 스킬트리 영역에서 구매 가능한 노드 (활성 상태, 녹색/파란색 테두리) 클릭
3. 상세 패널/구매 팝업 확인:
   - 노드 이름, 설명, 현재 레벨, 비용, 효과값 표시
   - "Lv X -> Lv X+1" 형식의 레벨 변화 텍스트
   - 보유 Bit와 비용 비교 (부족 시 빨간색)
4. 구매 버튼 클릭
5. **확인 사항**:
   - Bit 차감됨 (상단 바 수치 갱신)
   - 스킬 레벨 증가
   - 팝업 내용이 다음 레벨 정보로 갱신
   - 노드 시각적 상태 변경
6. 잠금 노드 (회색) 클릭 시:
   - 잠금 안내 패널 표시
   - 미충족 전제 조건 나열

### 3.3 전투 플로우 확인 (IG-01 ~ MN-08)

1. 허브에서 Stage 1 선택 후 "출격"
2. **인게임 진입 확인**:
   - Arrow 타워 3개 자동 배치됨
   - 상단 HUD 표시 (웨이브, Bit, HP 바)
   - Hub BGM -> Combat BGM 전환
3. **웨이브 진행 확인**:
   - 몬스터가 웨이포인트를 따라 이동
   - 타워가 사거리 내 몬스터를 자동 공격
   - 투사체가 발사되고 명중 시 몬스터 HP 감소
   - 몬스터 사망 시 Bit 드롭
   - 공격/사망 SFX 재생
4. **Cannon/Ice 타워 테스트** (해당 타워가 배치된 경우):
   - Cannon: 폭발 범위 내 복수 타격 확인
   - Ice: 피격 몬스터 이동속도 감소 확인
5. **Quick/Heavy 몬스터 테스트** (해당 웨이브 도달 시):
   - Quick: Bit Node보다 빠른 이동 확인
   - Heavy: 더 많은 타격을 버티는지 확인
6. **런 종료 확인**:
   - 클리어 또는 패배 시 런 종료 패널 슬라이드 업
   - 보상 정보 표시
   - Hub/재도전 버튼 동작

### 3.4 스프라이트 투명 배경 확인 (SP-01 ~ SP-03)

1. 인게임 진입
2. Game 뷰에서 각 오브젝트 관찰:
   - 타워 주변에 흰색/검은색/색상 사각형 배경 없는지 확인
   - 몬스터 주변에 불투명 배경 없는지 확인
   - 투사체가 비행 중 배경 없이 깨끗하게 표시되는지 확인
3. **판정 기준**: 스프라이트 외곽이 깨끗하게 투명 처리됨
4. **참고**: 87개 PNG에 대해 배경 투명화 처리 완료 (커밋 d4593d7)
