## 피드백

각 팀장들 업무 단위별로 커밋은 잘 하면서 진행하고 있는거야? 각각의 커밋이 팀장들 이름으로 될 수 있도록 환경 구축해서 각 팀장들의 업무들이 git에 잘 표현될 수 있도록 신경써줘.

Qa업무의 경우 더 미리 시작할 수 있는 부분들이 있었을 것 같아. 개발팀들의 업무가 다 끝나고나서야 전체 Qa를 시작하면 QA하는 동안 다른 팀장들은 다 기다리고만 있게 되잖아. QA팀장은 각 팀장들의 업무진척도와 상황을 확인하면서 먼저 QA할 수 있는 부분들은 먼저 진행하면서 병목현상을 최대한 줄여보면 좋겠어.

기획팀장은 전체적으로 하는 일이 없는 것 같은데, 프로그래밍 팀장은 로직 위주로 업무를 진행하고 몬스터의 숫자, 속도 이런 것들은 기획팀장이 진행할 수 있도록 별도 SO 로 잘 만들어서 기획팀에게 전달해줘. 전반적으로 프로그래밍 팀장에게 업무로드가 많이 걸려 있으니까 기획팀장은 수시로 미리미리 밸런스 작업이 레벨디자인 작업 할 수 있는 일들을 찾아서 프로그래밍 팀장 일들을 덜어줄 수 있도록 노력해. 물론 코딩을 하라는 건 아니야. 프로그래밍팀장을 통해서 수치정보로 빠진 것들을 만지면 돼.

프로그래밍 팀장과 UI팀장의 업무분배는 전반적으로 잘 되고 있는 것 같은데 어때? 문제가 있는게 있으면 말해줘.

그리고 스프린트 진행상황을 기록하는 PM 에이전트를 중간에 영입했으니 앞으로는 PM에이전트가 스프린트 모든 에이전트들과 소통하면서 스프린트 진행하는 일들을 기록하고 관리해줘. 관리라는 것은 병목 현상이 생긴 팀장들의 일을 분배할 수 있도록 스프린트 완료목표를 개발PD의 컨펌하에 조정해나가야 되는거야. 그러면서 목표했던 스프린트 일정안에 스프린트 완료빌드가 나올 수 있게 만드는 거지. 

각자 스프린트 진행하면서 병목 현상이 있었던 부분이나 조정되었으면 싶은 업무분배가 있었다면 자유롭게 말해줘.
