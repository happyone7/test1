## 프로세스/협업
git 사용방법이 일원화된 형태로 팀원 모두에게 공유되어야 해요. skill.sh등에서 git과 관련된 괜찮은 skill 문서가 있으면 받아서 공유했으면 좋겠어요.

특히 자기 업무가 끝났는데 커밋을 하지 않아서 이후 다른 사람 작업과 짬뽕되는 문제와, 다른 사람이 작업중인데 같이 작업해버리는 문제가 심각했어요. 다른 사람이 작업중인 파일은 건들지 않아야 합니다. 특히 씬이나 프리팹의 경우 프로그래밍 팀장과 UI팀장이 둘다 작업해야되는 경우가 많으므로 두분이 서로 각자가 수정중이지 않은지 확인하면서 겹치지 않도록 잘 관리해주세요.

skill-unity-folder-prefab-management 파일로 폴더와 프리팹에 관련된 규칙을 스킬로 정의해놨는데 다들 이걸 참고하면서 작업했나요? scene에 그냥 오브젝트가 놓여진 상태로 있고 프리팹화도 안되고, 오브젝트 구조도 제대로 적용되지 않은 것 같았어요. 오브젝트, 프리팹 구분과 정리를 하는 좋은 skill 정보를 찾아 적용해서 전 팀원에게 배포해주세요.
