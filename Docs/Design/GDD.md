## 1. 게임 개요

### 1.1 컨셉

**Nodebreaker TD**는 빠른 죽음과 영구 성장을 반복하며 도파민을 극대화하는 타워 디펜스 게임이다. Node라는 적들이 끊임없이 밀려오고, 플레이어는 타워를 배치해 처치하며 자원을 획득한다. 죽어도 성장은 유지되며, 반복 플레이를 통해 점점 더 강해지는 쾌감을 제공한다.

**핵심 키워드**: 초반 재미 극대화, 성장 도파민, 휘발성 즐거움, 빠른 죽음/재시도

### 1.3 레퍼런스

- **Nodebuster**: 핵심 레퍼런스. Node를 잡는 TD의 기본 구조
- **Vampire Survivors**: 빠른 성장 루프, 저가 게임의 도파민 설계
- **Idle/방치형 게임**: 영구 성장, 오프라인 보상 구조

| 30초 규칙 | 30초 안에 첫 죽음 또는 첫 성장을 경험해야 한다 |
| 30분 규칙 | 30분마다 체감 가능한 파워 도약이 있어야 한다 |
| 죽음은 보상 | 죽음이 실패가 아니라 성장의 과정으로 느껴져야 한다 |

## 2. 핵심 게임 루프

### 2.1 메인 루프

```
[플레이] -> [죽음] -> [영구 성장] -> [플레이] -> [죽음] -> [영구 성장] -> ...
              |                          |
              v                          v
         Node 처치 자원 누적        더 오래 생존, 더 많은 자원
              |
              +-- 스테이지 클리어 시 프리미엄 자원 추가 획득
```

### 2.2 인게임 루프 (1회 런)

```
1. 스테이지 진입 (기본 타워 자동 배치)
2. Node 웨이브 시작 (자동, 대기 없음)
3. 타워가 Node를 자동 공격
4. Node 처치 -> Bit(기본 자원) 획득
5. Bit로 타워 추가 배치 / 인게임 업그레이드
6. 더 강한 웨이브 도착
7. 기지 HP 0 -> 런 종료
8. 누적 Bit -> 영구 업그레이드 (Hub 화면)
9. 다시 1번으로
```

### 2.3 메타 루프 (스테이지 진행)

```
스테이지 1 반복 도전 -> 클리어 -> Core(프리미엄 자원) 획득
                                    |
                                    v
                        강력한 영구 업그레이드 / 신규 타워 해금
                                    |
                                    v
                        스테이지 2 도전 -> 클리어 -> ...
                                    |
                                    v
                        스테이지 10 클리어 = 엔딩
```

### 2.4 승리/패배 조건

- **런 종료 (죽음)**: 기지 HP가 0 이하. 패널티 없음. 획득한 Bit는 모두 유지.
- **스테이지 클리어**: 해당 스테이지의 최종 웨이브 돌파. Core 보상 획득.
- **게임 클리어 (엔딩)**: 스테이지 10 클리어.

설계 의도: 죽음에 패널티가 없으므로 플레이어는 부담 없이 재도전한다. "한 판만 더"를 유도하는 핵심 구조.

## 3. 성장 시스템

### 3.1 자원 종류

| 자원 | 이름 | 획득 방법 | 용도 | 영속성 |
|------|------|----------|------|--------|
| Bit | 기본 자원 | Node 처치 시 드롭 | 인게임 타워 배치/업그레이드, 런 종료 후 영구 업그레이드 | 영구 누적 |
| Core | 프리미엄 자원 | 스테이지 클리어 보상 | 강력한 영구 업그레이드, 신규 타워 해금 | 영구 누적 |
| Idle Bit | 방치 자원 | 오프라인/방치 시간 경과 | Bit와 동일하게 사용 | 접속 시 수령 |

설계 의도: Bit는 "플레이하면 항상 얻는" 기본 보상으로 죽음의 좌절감을 상쇄한다. Core는 "스테이지를 깨야만 얻는" 희귀 보상으로 클리어 동기를 부여한다.

### 3.2 스킬 트리 (Hub 영구 업그레이드)

런이 끝나면 Hub 화면에서 **스킬 트리**를 통해 영구 업그레이드를 구매한다. Nodebuster 스타일의 트리형 시스템으로, 화면 중앙에 시작 노드 1개가 있고, 이를 찍으면 사방으로 선택지가 뻗어나간다.

#### 트리 구조 원칙

- **시작**: 화면 중앙에 "Core Node" 1개. 이것을 찍으면 상하좌우 4방향으로 첫 번째 분기가 열림
- **분기**: 각 노드를 찍을 때마다 연결된 다음 노드들이 보임 (2~3갈래)
- **잠금**: 연결되지 않은 노드는 보이지 않음. 찍을수록 트리가 점점 펼쳐지는 탐험감
- **다중 자원**: 초반 노드는 Bit로 구매, 깊은 노드일수록 Core 필요
- **모든 노드 해금 가능**: 영구적 제한 없이 충분히 플레이하면 전부 찍을 수 있음 (타이밍 최적화가 핵심)

#### 트리 시각 레이아웃

```
                        [스폰율 증가]
                             |
                [Bit 획득+] ─ [공격속도+]
                     |            |
          [HP 회복] ─ [사거리+] ─ [크리티컬]
               |          |          |
    [Idle 수집기] ─ [★ Core Node ★] ─ [Cannon 해금]
               |          |          |
     [배속 해금] ─ [공격력+] ─ [기지 HP+]
                     |            |
              [Ice 해금] ─ [시작 Bit+]
                             |
                    [Lightning 해금]
                             |
                      [Laser 해금]
                             |
                      [Void 해금]
```

(실제 트리는 더 많은 노드와 분기를 가짐. 위는 핵심 구조 예시)

#### 트리 노드 목록

**Bit 노드 (기본 성장)**

| 노드 | 레벨당 효과 | 최대 레벨 | Bit 비용 공식 | 트리 위치 |
|------|-----------|---------|-------------|----------|
| 공격력 강화 | 타워 데미지 +10% | 20 | 50 * (1.3^Lv) | Core에서 1칸 |
| 공격속도 강화 | 타워 공속 +5% | 15 | 80 * (1.3^Lv) | Core에서 1칸 |
| 기지 HP 강화 | 기지 HP +5 | 20 | 40 * (1.25^Lv) | Core에서 1칸 |
| 사거리 강화 | 타워 사거리 +0.2 | 10 | 60 * (1.3^Lv) | Core에서 1칸 |
| Bit 획득량 | Node 처치 Bit +15% | 15 | 100 * (1.35^Lv) | 공격력에서 1칸 |
| 시작 Bit | 런 시작 Bit +30 | 10 | 120 * (1.4^Lv) | 기지 HP에서 1칸 |
| 스폰율 증가 | Node 스폰 +10% (= 더 많은 자원) | 10 | 90 * (1.3^Lv) | 공속에서 2칸 |
| HP 회복 | 기지 HP 초당 0.5 회복 | 5 | 200 * (1.4^Lv) | 사거리에서 2칸 |

**Core 노드 (프리미엄 성장 / 트리 깊은 곳)**

| 노드 | 효과 | Core 비용 | 전제 조건 | 설계 의도 |
|------|------|----------|----------|----------|
| 크리티컬 해금 | 15% 확률 2배 데미지 | 2 | 공격력 Lv3 | 타격감 대폭 증가 |
| Cannon 해금 | Cannon Tower 사용 가능 | 3 | 크리티컬 해금 | AoE 전략 추가 |
| Ice 해금 | Ice Tower 사용 가능 | 5 | 공속 Lv5 | 감속 유틸리티 |
| Lightning 해금 | Lightning Tower 사용 가능 | 8 | Cannon + Ice 해금 | 체인 공격 |
| Laser 해금 | Laser Tower 사용 가능 | 12 | Lightning 해금 | 관통 지속 공격 |
| Void 해금 | Void Tower 사용 가능 | 18 | Laser 해금 | 최종 타워 |
| Idle 수집기 | 오프라인 Bit 생성 | 3 | 사거리 Lv3 | 방치형 요소 |
| 배속 해금 | 2배속/3배속 모드 | 2 | 기지 HP Lv2 | 반복 가속 |
| 타워 슬롯 확장 | 배치 칸 +4 (최대 3회) | 6/회 | Cannon 해금 | 전략 폭 확대 |

설계 의도: 트리 시스템은 "다음에 뭘 찍을까?"라는 탐험 동기를 제공한다. 전통적인 목록형 업그레이드 대비 (1) 찍을수록 펼쳐지는 시각적 쾌감, (2) 경로 선택에 따른 빌드 다양성, (3) "이 방향으로 더 가면 뭐가 나올까?" 기대감을 준다.

### 3.3 타워 배치/합성 시스템 (인게임)

런 중 타워를 획득하고 배치하는 시스템. 스킬 트리에서 해금한 타워만 사용 가능.

#### 타워 획득

- 웨이브 클리어 시 보상으로 랜덤 타워 1기 지급 (해금된 타워 중 랜덤)
- Bit로 타워 직접 구매 가능 (하단 상점)
- 획득한 타워는 하단 **타워 인벤토리**에 쌓임

#### 타워 배치

1. 하단 인벤토리에 있는 타워를 **드래그**하여 맵의 빈 격자 칸에 놓으면 배치
2. 배치된 타워는 다음 런에서도 **자동 유지** (죽어도 배치 유지)
3. 새로 획득한 타워만 하단 인벤토리에 표시되어 추가 배치 가능

#### 타워 합성 (레벨업)

- **같은 타워 위에 드래그**: 같은 종류의 타워를 이미 배치된 같은 타워 위에 올리면 합성
- 합성 시 레벨 +1 (최대 Lv5)
- 합성 이펙트: 번쩍이는 강화 연출 + 수치 상승 팝업

```
예시:
Arrow Lv1 (배치됨) + Arrow Lv1 (인벤토리에서 드래그) = Arrow Lv2
Arrow Lv2 (배치됨) + Arrow Lv1 (드래그) = Arrow Lv3
```

#### 타워 판매

- 배치된 타워를 길게 클릭 → 판매 (Bit 70% 환불)
- 또는 인벤토리 밖으로 드래그하여 버리기

설계 의도: 드래그 배치 + 합성 시스템은 (1) 직관적인 조작감, (2) 같은 타워를 모아서 합치는 수집+성장의 쾌감, (3) "이 타워를 여기에 쓸까 합성에 쓸까?"라는 전술적 선택을 제공한다. 배치 유지로 매 런마다 재배치하는 번거로움을 제거.

### 3.4 성장 체감 설계

| 시점 | 영구 공격력 배율 (예상) | 체감 |
|------|---------------------|------|
| 첫 플레이 | x1.0 | 기본. 30초~1분에 죽음 |
| 5회차 | x1.5 | 눈에 띄게 더 오래 생존 |
| 15회차 | x2.5 | 스테이지 1 클리어 가능 |
| 30회차 | x4.0 | 스테이지 3~4 진행 |
| 50회차 | x8.0 | 폭발적 성장 구간. 화면이 이펙트로 가득 |
| 80회차~ | x15.0+ | 후반 스테이지. 대규모 Node 학살 |

설계 의도: 지수적 성장 곡선으로 "방금 전보다 확실히 강해졌다"를 매번 느끼게 한다. 특히 x4.0~x8.0 구간(1~2시간차)이 가장 재미있는 구간으로 설계.

## 4. 스테이지/웨이브 시스템

### 4.1 스테이지 구조

게임은 10개의 스테이지로 구성된다. 각 스테이지는 고유한 맵, Node 구성, 시각적 테마를 가진다.

| 스테이지 | 이름 | 웨이브 수 | 테마 | 해금 조건 | 클리어 보상 (Core) | 설계 의도 |
|---------|------|---------|------|----------|-----------------|----------|
| 1 | Data Stream | 5 | 녹색 회로 | 시작 | 2 | 학습용. 기본 메카닉 체득 |
| 2 | Memory Block | 7 | 파란 격자 | 스테이지 1 클리어 | 3 | Cannon 해금으로 AoE 경험 |
| 3 | Cache Layer | 8 | 보라 네온 | 스테이지 2 클리어 | 4 | Ice 해금. 감속 전략 학습 |
| 4 | Pipeline | 10 | 주황 회로 | 스테이지 3 클리어 | 5 | 다중 경로 도입 |
| 5 | Processor Core | 10 | 붉은 네온 | 스테이지 4 클리어 | 6 | Lightning 해금. 중반 전환점 |
| 6 | Bus Network | 12 | 백색+청색 | 스테이지 5 클리어 | 7 | 대규모 물량전 |
| 7 | Kernel Space | 12 | 짙은 보라 | 스테이지 6 클리어 | 9 | Laser 해금. 후반 시작 |
| 8 | Overflow Zone | 15 | 붉은+검은 | 스테이지 7 클리어 | 11 | 극한 난이도 시작 |
| 9 | Root Access | 15 | 금색 회로 | 스테이지 8 클리어 | 14 | Void 해금. 최종 준비 |
| 10 | Kernel Panic | 20 | 전 테마 혼합 | 스테이지 9 클리어 | 엔딩 | 최종 보스. 모든 요소 총출동 |

### 4.2 웨이브 규칙

- **자동 시작**: 웨이브 간 대기 없음. 이전 웨이브 종료 후 3초 뒤 자동 시작.
- **밀도 증가**: 웨이브가 진행될수록 스폰 간격이 짧아지고 Node 수가 증가.
- **클리어 판정**: 마지막 웨이브의 모든 Node를 처치하거나 통과시키면 스테이지 클리어.
- **죽음 판정**: 기지 HP가 0 이하가 되면 즉시 런 종료.

설계 의도: 대기 시간을 최소화하여 "빠른 죽음 -> 빠른 재시도" 루프를 유지한다. 3초 간격은 타워를 1~2기 추가 배치할 수 있는 최소한의 여유.

### 4.3 스테이지 1 상세 (기준 예시)

| 웨이브 | Node 구성 | 스폰 간격(초) | 설계 의도 |
|--------|----------|-------------|----------|
| 1 | Bit Node x5 | 1.2 | 기본 학습. "Node가 오고, 타워가 잡는다" |
| 2 | Bit Node x8 | 1.0 | 수량 증가. 타워 추가 배치 유도 |
| 3 | Bit Node x6, Quick Node x3 | 0.8 | 빠른 적 등장. 긴장감 추가 |
| 4 | Bit Node x10, Quick Node x5 | 0.7 | 물량 압박. 영구 업그레이드 없으면 여기서 사망 가능 |
| 5 | Bit Node x8, Quick Node x4, Heavy Node x2 | 0.6 | 최종 웨이브. 첫 클리어 시 큰 성취감 |

초기 상태(영구 업그레이드 0)에서는 웨이브 2~3에서 죽는 것이 정상. 이것이 "성장해야 한다"는 동기 부여.

## 5. 타워 시스템

### 5.1 타워 공통 동작

1. 사거리 내 가장 가까운 Node를 자동 타겟팅
2. 공격 쿨타임마다 투사체/효과 발사
3. 인게임 Bit로 레벨업 가능 (런 내 최대 Lv5)
4. 영구 업그레이드 배율이 곱연산으로 적용됨

### 5.2 타워 목록 및 수치

#### T01 - Arrow Tower (기본 타워, 시작 해금)

단일 대상 고속 공격. 가장 먼저 쓰고 가장 오래 쓰는 범용 타워.

| 스탯 | Lv1 | Lv2 | Lv3 | Lv4 | Lv5 |
|------|-----|-----|-----|-----|-----|
| 인게임 비용 (Bit) | 30 | 25 | 35 | 50 | 70 |
| 데미지 | 8 | 12 | 18 | 26 | 38 |
| 공격속도 (초) | 0.8 | 0.7 | 0.6 | 0.5 | 0.4 |
| 사거리 (유닛) | 3.0 | 3.2 | 3.4 | 3.6 | 4.0 |
| DPS (기본) | 10.0 | 17.1 | 30.0 | 52.0 | 95.0 |

설계 의도: Lv1은 약하게 느껴지지만 영구 업그레이드와 곱연산되면 후반에도 유효하다. 공격속도 스케일링이 좋아서 "빠르게 쏘는 맛"을 제공.

#### T02 - Cannon Tower (AoE, 스테이지 2 클리어 후 Core로 해금)

범위 폭발 공격. 밀집된 Node 무리에 강하다.

| 스탯 | Lv1 | Lv2 | Lv3 | Lv4 | Lv5 |
|------|-----|-----|-----|-----|-----|
| 인게임 비용 (Bit) | 60 | 50 | 70 | 100 | 140 |
| 데미지 | 20 | 35 | 55 | 80 | 120 |
| 공격속도 (초) | 1.5 | 1.3 | 1.1 | 0.9 | 0.7 |
| 사거리 (유닛) | 2.5 | 2.7 | 3.0 | 3.3 | 3.6 |
| 폭발 범위 (유닛) | 1.0 | 1.1 | 1.2 | 1.4 | 1.6 |
| DPS (단일) | 13.3 | 26.9 | 50.0 | 88.9 | 171.4 |

설계 의도: 단일 DPS는 Arrow보다 낮지만, 3~5마리를 동시에 타격하면 총 효율이 압도적. 물량 웨이브에서 필수.

#### T03 - Ice Tower (감속, 스테이지 3 클리어 후 Core로 해금)

데미지보다 감속 유틸리티에 특화. 다른 타워의 공격 시간을 벌어준다.

| 스탯 | Lv1 | Lv2 | Lv3 | Lv4 | Lv5 |
|------|-----|-----|-----|-----|-----|
| 인게임 비용 (Bit) | 45 | 35 | 50 | 70 | 100 |
| 데미지 | 4 | 7 | 12 | 18 | 28 |
| 공격속도 (초) | 1.0 | 0.9 | 0.8 | 0.7 | 0.6 |
| 사거리 (유닛) | 3.0 | 3.2 | 3.5 | 3.8 | 4.2 |
| 감속률 | 25% | 35% | 45% | 55% | 65% |
| 감속 지속 (초) | 1.5 | 1.8 | 2.0 | 2.3 | 2.5 |

설계 의도: 단독으로는 약하지만, Arrow/Cannon 앞에 배치하면 시너지가 극대화. "타워 조합"이라는 전략적 재미를 열어주는 타워.

#### T04 - Lightning Tower (체인, 스테이지 5 클리어 후 Core로 해금)

공격이 적에게 맞은 뒤 주변 적에게 체인으로 전이.

| 스탯 | Lv1 | Lv2 | Lv3 | Lv4 | Lv5 |
|------|-----|-----|-----|-----|-----|
| 인게임 비용 (Bit) | 80 | 65 | 90 | 130 | 180 |
| 데미지 | 12 | 20 | 32 | 48 | 70 |

| 사거리 (유닛) | 3.0 | 3.3 | 3.5 | 3.8 | 4.0 |
| 체인 횟수 | 2 | 3 | 4 | 5 | 6 |
| 체인 데미지 감소 | 30% | 25% | 20% | 15% | 10% |

설계 의도: 중반 해금. 체인으로 다수를 동시 처리하면서 시각적으로도 화려하다. "강해졌다"는 느낌을 주는 핵심 타워.

#### T05 - Laser Tower (관통 지속, 스테이지 7 클리어 후 Core로 해금)

레이저 빔을 발사하여 일직선상의 모든 적에게 지속 데미지.

| 스탯 | Lv1 | Lv2 | Lv3 | Lv4 | Lv5 |
|------|-----|-----|-----|-----|-----|
| 인게임 비용 (Bit) | 120 | 100 | 140 | 200 | 280 |
| 초당 데미지 | 25 | 40 | 60 | 90 | 140 |
| 빔 지속 (초) | 2.0 | 2.2 | 2.5 | 2.8 | 3.0 |
| 쿨다운 (초) | 3.0 | 2.8 | 2.5 | 2.2 | 2.0 |
| 사거리 (유닛) | 4.0 | 4.3 | 4.5 | 4.8 | 5.0 |
| 관통 수 | 3 | 4 | 5 | 7 | 무제한 |

설계 의도: 후반 타워. 일직선 배치 시 경로를 따라 다수의 Node를 관통하는 쾌감. 시각적으로도 매우 강렬.

#### T06 - Void Tower (최종, 스테이지 9 클리어 후 Core로 해금)

범위 내 모든 적에게 지속적으로 데미지. 최종 타워.

| 스탯 | Lv1 | Lv2 | Lv3 | Lv4 | Lv5 |
|------|-----|-----|-----|-----|-----|
| 인게임 비용 (Bit) | 150 | 130 | 180 | 250 | 350 |
| 초당 데미지 (범위 내 전체) | 15 | 25 | 40 | 65 | 100 |
| 효과 범위 (유닛) | 2.0 | 2.3 | 2.6 | 3.0 | 3.5 |
| 사거리 | 자기 주변 (범위=사거리) | - | - | - | - |
| 추가 효과 | - | - | 방어력 무시 | +감속 10% | +감속 20% |

설계 의도: 경로 한가운데 배치하면 지나가는 모든 Node에 데미지. 영구 업그레이드와 조합하면 "걸어가는 것만으로 Node가 녹는" 쾌감을 제공. 최종 타워답게 비싸지만 압도적.

### 5.3 타워 해금 (스킬 트리)

타워 해금은 Hub 스킬 트리의 Core 노드를 찍어야 한다. 트리에서 해당 노드까지 도달하려면 전제 노드들을 먼저 찍어야 함.

```
Arrow Tower: 시작 시 기본 제공
Cannon Tower: 트리에서 크리티컬 해금 → Cannon 해금 노드 (Core 3)
Ice Tower: 트리에서 공속 Lv5 → Ice 해금 노드 (Core 5)
Lightning Tower: Cannon + Ice 해금 → Lightning 해금 노드 (Core 8)
Laser Tower: Lightning 해금 → Laser 해금 노드 (Core 12)
Void Tower: Laser 해금 → Void 해금 노드 (Core 18)
```

설계 의도: 타워 해금이 트리의 깊은 곳에 위치하여, 그곳까지 찍어가는 과정 자체가 성장. 경로 위에 있는 Bit 노드들도 자연스럽게 찍게 되어 전체적인 파워 상승.

## 6. 적(Node) 시스템

### 6.1 Node 공통 동작

1. 스폰 포인트에서 생성되어 정해진 경로를 따라 기지로 이동
2. HP가 0이 되면 파괴, Bit 드롭 (화면에 숫자 팝업)
3. 기지에 도달하면 기지 HP를 깎고 사라짐
4. 스테이지가 올라갈수록 HP/속도에 스케일링 배율 적용

### 6.2 Node 목록

| ID | 이름 | 특성 | 등장 스테이지 |
|----|------|------|-------------|
| N01 | Bit Node | 기본. 느리고 약함 | 전 스테이지 |
| N02 | Quick Node | 빠른 이동속도 | 스테이지 1~ |
| N03 | Heavy Node | 높은 HP | 스테이지 1~ |
| N04 | Shield Node | 고정 방어력 보유 | 스테이지 3~ |
| N05 | Swarm Node | 매우 약하지만 대량 스폰 | 스테이지 4~ |
| N06 | Regen Node | HP 자동 회복 | 스테이지 5~ |
| N07 | Phase Node | 주기적으로 무적 | 스테이지 6~ |
| N08 | Split Node | 죽으면 소형 Node 2마리로 분열 | 스테이지 7~ |
| N09 | Boss Node | 스테이지 최종 웨이브 등장. 극히 높은 HP | 스테이지별 1체 |

### 6.3 Node 기본 수치 (스테이지 1 기준)

| ID | 이름 | HP | 이동속도 | 방어력 | 기지 데미지 | Bit 드롭 |
|----|------|-----|---------|--------|-----------|---------|
| N01 | Bit Node | 20 | 1.5 | 0 | 1 | 3 |
| N02 | Quick Node | 12 | 3.0 | 0 | 1 | 4 |
| N03 | Heavy Node | 80 | 0.8 | 0 | 2 | 8 |
| N04 | Shield Node | 40 | 1.2 | 5 | 1 | 6 |
| N05 | Swarm Node | 5 | 2.0 | 0 | 1 | 1 |
| N06 | Regen Node | 50 | 1.0 | 0 | 1 | 7 |
| N07 | Phase Node | 35 | 1.5 | 0 | 1 | 8 |
| N08 | Split Node | 30 | 1.3 | 0 | 1 (분열체: 1) | 5 (분열체: 2) |
| N09 | Boss Node | 300 | 0.5 | 3 | 5 | 50 |

- **이동속도**: 유닛/초. 1.5가 기준.
- **방어력**: 받는 데미지에서 고정 차감 (최소 1).
- **Regen Node**: 초당 HP의 5% 회복.
- **Phase Node**: 2초 이동 후 1초 무적 (반복).

### 6.4 스테이지별 스케일링

| 스테이지 | HP 배율 | 속도 배율 | Bit 드롭 배율 |

### 6.5 Node 설계 의도

| Node | 만드는 상황 | 대응 전략 |
|------|-----------|----------|
| Bit Node | 기본 자원 공급원 | Arrow Tower로 충분 |

| Shield Node | 약한 공격 무효화 | 고데미지 타워 (Cannon, Laser) |
| Swarm Node | 대량으로 밀려옴 -> AoE 필요 | Cannon, Lightning, Void |
| Regen Node | 빨리 안 잡으면 회복됨 | 집중 화력 (고DPS) |
| Phase Node | 타이밍 맞춰야 함 | 지속 데미지 타워 (Laser, Void) |
| Split Node | 잡아도 2마리 추가 | AoE로 분열체까지 처리 |
| Boss Node | 모든 자원 투입 | 전체 타워 조합 |

## 7. 경제 시스템

### 7.1 Bit 경제 (인게임)

#### Bit 수입원

| 수입원 | 양 | 비고 |
|--------|-----|------|
| Node 처치 | Node별 기본 드롭 * 스테이지 배율 * 영구 수입 배율 | 주 수입원 |
| 웨이브 클리어 보너스 | 10 + (웨이브 번호 * 5) | 웨이브 완료 시 |
| 콤보 보너스 | 3초 내 5마리 이상 연속 처치 시 +50% Bit | 스킬 보상 |

#### Bit 지출처 (인게임)

| 지출 | 비용 범위 |
|------|----------|
| 타워 배치 | 30~150 Bit (타워별) |
| 타워 레벨업 | 점진적 증가 |
| 타워 판매 | 70% 환불 |

#### Bit 지출처 (Hub 영구 업그레이드)

- 3.2절 Bit 업그레이드 표 참조
- 비용이 지수적으로 증가하여 자연스러운 진행 속도 조절

### 7.2 Core 경제

| 획득 | 양 |
|------|-----|
| 스테이지 첫 클리어 | 스테이지별 고정량 (2~14 Core) |
| 스테이지 재클리어 | 첫 클리어의 30% (반올림) |

| 지출 | 비용 범위 |
|------|----------|
| 신규 타워 해금 | 3~18 Core |
| 프리미엄 업그레이드 | 2~6 Core |

#### Core 총량 검증

| 시점 | 누적 Core (첫 클리어만) | 주요 구매 |
|------|----------------------|----------|
| St.1 클리어 | 2 | 크리티컬/Idle/배속 중 택1 |
| St.2 클리어 | 5 | Cannon 해금 (3) |
| St.3 클리어 | 9 | Ice 해금 (5) + 여유 1 |
| St.5 클리어 | 20 | Lightning 해금 (8) |
| St.7 클리어 | 36 | Laser 해금 (12) |
| St.9 클리어 | 61 | Void 해금 (18) |

설계 의도: Core는 항상 약간 부족하게 설계. "이번에 뭘 먼저 살까?" 고민이 선택의 재미. 재클리어 30% 보상으로 부족분을 메울 수 있지만 효율이 떨어져, 앞으로 나아가는 것이 항상 최선.

### 7.3 경제 밸런스 의도

```
초반 (0~30분):
  - 1회 런에서 Bit 50~200 획득
  - 영구 업그레이드 1~2개 가능
  - "죽었지만 뭔가 샀다" -> 긍정적 루프

중반 (30분~2시간):
  - 1회 런에서 Bit 500~3000 획득
  - 스테이지 클리어로 Core 획득
  - 신규 타워 해금 -> 전략 확장 -> 더 많은 Bit

후반 (2~5시간):
  - 1회 런에서 Bit 10000+ 획득
  - 다수의 영구 업그레이드 맥스
  - 스케일의 폭발적 증가
```

## 8. 도파민 설계

### 8.1 피드백 계층

| 계층 | 트리거 | 피드백 | 빈도 |
|------|--------|--------|------|
| 마이크로 | Node 1마리 처치 | 파괴 이펙트 + Bit 숫자 팝업 + 경쾌한 효과음 | 매초 |
| 소형 | 웨이브 클리어 | 화면 플래시 + "WAVE CLEAR" 텍스트 + 보너스 Bit | 30초~1분 |
| 중형 | 영구 업그레이드 구매 | 업그레이드 이펙트 + 수치 비교(Before/After) + "강해진 느낌" 연출 | 런 종료마다 |
| 대형 | 스테이지 클리어 | Core 획득 연출 + 새 스테이지 해금 + 신규 타워 해금 가능 | 30분~1시간 |
| 특대 | 신규 타워 첫 사용 | 전용 연출 + 타워 소개 팝업 + 첫 공격 시 슬로우모션 | 타워 해금 시 |

### 8.2 Node 처치 이펙트 스케일링

| 상황 | 이펙트 |
|------|--------|
| 일반 처치 | 작은 파티클 + "+3" 텍스트 |
| 연속 처치 (콤보) | 파티클 크기 증가 + 콤보 카운터 표시 + "COMBO x5!" |
| 대량 동시 처치 (AoE) | 큰 폭발 이펙트 + 화면 흔들림(미세) + 합산 Bit 표시 |
| Boss 처치 | 전체 화면 이펙트 + 슬로우모션 1초 + 대형 Bit 아이콘 |
| 스테이지 클리어 | 전체 화면 연출 + Core 등장 애니메이션 |

설계 의도: 성장할수록 이펙트가 화려해진다. 처음에는 작은 파티클이 뜨지만, 후반에는 화면이 이펙트로 가득 차는 경험. "내가 이렇게 강해졌구나"를 시각적으로 체감.

### 8.3 보상 타이밍 설계

```
0초: 게임 시작
3초: 첫 Node 등장
5초: 첫 Node 처치 -> "+3 Bit" (첫 보상)
15초: 타워 1기 추가 배치 가능 (첫 성장)
30초~1분: 첫 죽음 -> Hub 진입 -> "영구 업그레이드 가능!" (핵심 후크)
1분: 두 번째 런 시작. 영구 업그레이드 덕에 더 오래 생존 -> "효과가 있네!" (확인)
2분: 웨이브 3~4 도달. 새로운 Node 타입 등장 -> "이건 어떻게 대응하지?" (호기심)
5분: 여러 번의 런으로 안정적 성장. 스테이지 1 클리어 가능성 체감 -> "곧 깰 수 있겠다" (목표)
```

## 9. 방치형 요소

### 9.1 Idle Bit 수집기

스테이지 1 클리어 후 Core로 해금. 게임을 종료해도 시간이 지나면 Bit가 쌓인다.

| 항목 | 값 | 업그레이드 |

```
|                       |                                    |

|               |          |          |                      |
|   [Idle수집] - [★CENTER★] - [Cannon]                       |
|               |          |          |                      |
|    [배속] - [공격력+] - [기지HP+]                            |
|                       |                                    |
|                   [Ice 해금]                                |
|                                                            |
|    (우클릭 드래그로 트리 이동 / 스크롤로 줌인아웃)              |
|                                                            |
+----------------------------------------------------------+
| [스테이지 선택 ▼]                            [설정] [종료]  |
+----------------------------------------------------------+
```

- **트리 영역**: 화면 대부분을 차지. 노드를 클릭하면 구매 팝업 (비용, 효과, Before/After)
- **찍은 노드**: 밝게 빛남. 연결선이 활성화
- **찍을 수 있는 노드**: 테두리 반짝임. 비용 표시
- **잠긴 노드**: 어둡게 처리. 전제 노드를 찍어야 보임
- **아직 안 보이는 노드**: 완전히 숨김 (트리가 점점 펼쳐지는 탐험감)
- **하단**: 스테이지 선택 드롭다운 (해금된 스테이지 목록)

#### 노드 구매 팝업

### 10.2 화면 구성: 인게임 (런 중)

```
+----------------------------------------------------------+
|  Wave: 3/5     Bit: 85     Base HP: ████████░░ 16/20     |
+----------------------------------------------------------+
|                                                            |
|                                                            |
|                     [게임 맵]                               |
|          (배치된 타워들이 Node를 공격하는 화면)                |
|          (빈 격자 칸이 배치 가능 위치로 표시됨)                |
|                                                            |
|                                                            |
+----------------------------------------------------------+
| 타워 인벤토리:                                              |
| [Arrow] [Arrow] [Cannon] [+구매]  | [x1] [x2] [x3] [⏸]  |
+----------------------------------------------------------+
```

- **상단 바**: 현재 웨이브, 보유 Bit, 기지 HP (체력바)
- **게임 맵**: 이미 배치된 타워 + 이동하는 Node들. 빈 격자가 배치 가능 슬롯
- **하단 - 타워 인벤토리**: 현재 보유 중인 미배치 타워들. 드래그로 배치
- **[+구매]**: Bit로 타워 즉시 구매 (해금된 타워 중 선택)
- **배속 + 일시정지**: 우측 하단
- 기지 HP 낮을 때 화면 테두리 붉은색 경고

### 10.3 타워 배치/합성 UX

#### 신규 배치

1. 하단 인벤토리에서 타워 아이콘을 **드래그**
2. 맵의 빈 격자 위로 가져가면 초록색 하이라이트
3. 손을 떼면 즉시 배치 (확인 없음 - 속도 우선)

#### 합성 (레벨업)

1. 인벤토리에서 타워를 **드래그**
2. 맵에 이미 배치된 **같은 종류** 타워 위로 가져가면 노란색 하이라이트 + "LV UP!" 표시
3. 손을 떼면 합성 → 레벨 +1 + 강화 이펙트 + 수치 팝업

```
합성 예시:
[Arrow Lv1] 위에 [Arrow] 드래그 → [Arrow Lv2] + 번쩍 이펙트
[Arrow Lv2] 위에 [Arrow] 드래그 → [Arrow Lv3] + 더 큰 이펙트
```

#### 타워 판매

- 배치된 타워를 **길게 클릭** → 판매 확인 → Bit 70% 환불
- 또는 배치된 타워를 드래그하여 **인벤토리 밖(화면 아래)으로** 끌면 판매

#### 배치 유지 규칙

- 런 종료(죽음) 후 다시 시작해도 **이전에 배치한 타워는 그대로 유지**
- 새로 획득/구매한 타워만 인벤토리에 표시되어 추가 배치 가능
- 합성으로 레벨업한 타워도 유지

## 12. 성장 단계별 플레이어 경험 타임라인

### 12.1 Phase 1: 온보딩 + 첫 도파민 (0~2분)

**목표**: "Node가 오고, 타워가 잡고, 자원을 얻는다"를 직관적으로 체득

| 시간 | 이벤트 | 플레이어 경험 | 감정 |
|------|--------|-------------|------|
| 0:00 | 게임 첫 시작. Hub 스킵, 바로 스테이지 1 진입 | "바로 시작이네?" | 호기심 |
| 0:03 | Arrow Tower 1기 자동 배치됨. 첫 Node 등장 | "뭔가 온다" | 긴장 |
| 0:05 | 타워가 Node를 자동 공격. Node 파괴. "+3 Bit" | "오, 잡힌다!" | 만족 |
| 0:15 | Bit 축적. "타워를 배치하세요" 안내 | "하나 더 놓아볼까" | 능동적 참여 |
| 0:30 | 웨이브 2~3. Node 수 증가. 기지 HP 감소 시작 | "좀 힘든데?" | 긴장 |
| 0:45~1:00 | 첫 죽음. "RUN COMPLETE" 화면 | "아, 죽었네" | 약간의 아쉬움 |
| 1:00 | Hub 진입. "영구 업그레이드 가능!" 안내 | "이걸로 강해지는 건가?" | 기대 |
| 1:10 | 기본 공격력 Lv1 구매. Before/After 표시 | "데미지가 올랐다" | 성취감 |
| 1:15 | 즉시 재시도. 같은 스테이지 진입 | "이번엔 더 가보자" | 동기부여 |
| 1:30 | 이전보다 확실히 더 오래 생존 | "확실히 강해졌다!" | 핵심 후크 확인 |
| 2:00 | 2~3회 런 완료. 성장 루프 이해 | 메인 루프 체득 완료 | 몰입 |

설계 의도: 첫 2분 안에 (1) 기본 메카닉, (2) 죽음, (3) 성장, (4) 재도전이라는 전체 루프를 경험시킨다. "빠른 죽음"이 핵심이므로 초기 밸런스는 의도적으로 빡빡하게.

### 12.2 Phase 2: 첫 성장 체감 (2~5분)

**목표**: "성장하면 더 강해지고, 더 강해지면 더 많이 번다"를 체감

| 시간 | 이벤트 | 플레이어 경험 |
|------|--------|-------------|
| 2~3분 | 영구 업그레이드 2~3개 구매 | 기초 스탯이 눈에 띄게 증가 |
| 3~4분 | 웨이브 3~4까지 안정적 도달 | 이전에 죽던 곳을 거뜬히 통과 |
| 4~5분 | 인게임에서 타워 2~3기 추가 배치 가능 | 자원 순환 루프 체감 |
| 5분 | 웨이브 4~5에서 죽음 → 더 많은 Bit 획득 | "점점 더 많이 벌린다" |

### 12.3 Phase 3: 메인 루프 확립 + 첫 클리어 (~30분)

**목표**: 반복 플레이의 리듬 확립, 스테이지 1 첫 클리어

| 시간 | 이벤트 | 플레이어 경험 |
|------|--------|-------------|
| 5~15분 | 빠른 런 반복 (1~2분/런) | 죽음-성장-재시도가 자연스러운 리듬 |
| 15~20분 | 영구 업그레이드 다수 구매, 스테이지 1 클리어 근접 | "이번엔 깰 수 있을 것 같다" |
| 20~30분 | 스테이지 1 클리어! Core 2개 획득 | 큰 성취감. 신규 콘텐츠 해금 |
| 30분 | 크리티컬/배속/Idle 중 선택 → 첫 Core 구매 | "뭘 살지 고민된다" = 좋은 고민 |

### 12.4 Phase 4: 폭발적 성장 (~1시간)

**목표**: 가장 재미있는 구간. 파워 커브의 가파른 상승.

| 시간 | 이벤트 | 플레이어 경험 |
|------|--------|-------------|
| 30~40분 | 스테이지 2 도전. 새로운 비주얼. Cannon Tower 해금 | "새 타워다!" + "새 스테이지다!" |
| 40~50분 | AoE 타워로 Node 대량 처치 시작 | 화면이 이펙트로 화려해짐 |
| 50분~1시간 | 스테이지 2~3 클리어. Ice Tower 해금 | 타워 조합 전략이 열림 |
| 1시간 | 영구 업그레이드 다수 축적. 전투/자원 규모 대폭 증가 | "전보다 훨씬 강하다" |

### 12.5 Phase 5: 콘텐츠 해금과 빌드 다양성 (~3시간)

**목표**: 새로운 타워, 새로운 Node, 새로운 스테이지의 연속 해금

| 시간 | 이벤트 |
|------|--------|
| 1~1.5시간 | 스테이지 4~5 진행. Lightning Tower 해금 |
| 1.5~2시간 | 체인 공격으로 화면이 번개로 가득. 스케일 감각 |

## 부록 B: 밸런스 설계 근거

### B.1 초반 밸런스 (스테이지 1, 영구 업그레이드 0)

- 기지 HP: 20
- Arrow Tower Lv1 DPS: 10.0
- Bit Node HP: 20, 스폰 간격 1.2초
- Arrow 1기로 Bit Node 1마리 처치: 2.0초. 스폰 간격보다 길다.
- 웨이브 1 (5마리): 1기로는 1~2마리 놓침 -> 기지 HP 18~19
- 웨이브 2 (8마리): 타워 추가 배치 해도 밀림 -> 기지 HP 10~14
- 웨이브 3 (Quick Node 등장): 높은 확률로 사망
- **결론**: 초기 상태에서 웨이브 2~3에 죽는 것이 정상. 이것이 "성장해야 한다"는 핵심 동기.

### B.2 첫 클리어 밸런스 (스테이지 1, 약 15~20회 런 후)

- 예상 영구 업그레이드: 공격력 Lv3(+30%), 공속 Lv2(+10%), 체력 Lv2(+10 HP)
- Arrow DPS 보정: 10.0 * 1.3 * 1.1 = 14.3
- 기지 HP: 30
- 이 수준이면 웨이브 5까지 안정적 클리어 가능

### B.3 후반 스케일링 검증 (스테이지 7)

- Node HP 배율: x22.0 -> Bit Node HP: 440
- 영구 공격력 예상: +200% (x3.0)
- Arrow Lv5 DPS: 95.0 * 3.0 = 285
- Bit Node 처치 시간: 1.5초 -> 문제 없음
- Boss Node HP: 300 * 22.0 = 6600 -> 다수 타워 조합 필요 -> 적절한 난이도

### B.4 경제 밸런스 검증

- 스테이지 1 런 평균 Bit 획득: 50~100 (죽는 시점에 따라)
- 영구 업그레이드 Lv1 평균 비용: 약 75
- 1~2회 런당 1개 업그레이드 = 체감 가능한 성장 속도
- 15~20회 런 (약 20~30분)에 스테이지 1 클리어 = 목표 타임라인과 부합

## 부록 D: 용어 정리

| 용어 | 설명 |
|------|------|
| Node | 적 유닛의 총칭. 데이터 네트워크의 노드 비유 |
| Bit | 기본 자원. Node 처치로 획득 |
| Core | 프리미엄 자원. 스테이지 클리어로 획득 |
| 런 (Run) | 1회 플레이. 스테이지 진입부터 죽음/클리어까지 |
| Hub | 런 사이의 메뉴 화면. 영구 업그레이드 구매 |
| 스킬 트리 | Hub의 트리형 영구 업그레이드 시스템. 중앙에서 사방으로 뻗어남 |
| 노드 (트리) | 스킬 트리의 개별 업그레이드 슬롯 |
| 합성 | 같은 타워를 겹쳐서 레벨업하는 시스템 |
| 타워 인벤토리 | 인게임 하단의 미배치 타워 목록 |
| 웨이브 | 한 스테이지 내에서 Node가 밀려오는 단위 |
| 스테이지 | 여러 웨이브로 구성된 하나의 맵/레벨 |
