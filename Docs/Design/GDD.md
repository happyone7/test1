## 1. 게임 개요

### 1.1 컨셉

**Nodebreaker TD**는 빠른 죽음과 영구 성장을 반복하며 도파민을 극대화하는 타워 디펜스 게임이다. Node라는 적들이 끊임없이 밀려오고, 플레이어는 타워를 배치해 처치하며 자원을 획득한다. 죽어도 성장은 유지되며, 반복 플레이를 통해 점점 더 강해지는 쾌감을 제공한다.

**핵심 키워드**: 초반 재미 극대화, 성장 도파민, 휘발성 즐거움, 빠른 죽음/재시도

**성장 구조**: 순수 영구 성장(런마다 동일 웨이브, Bit로 영구 성장) + 런 내 랜덤 요소(보물상자 선택지)

## 2. 핵심 게임 루프

### 2.1 메인 루프

```
[플레이] -> [죽음] -> [영구 성장] -> [플레이] -> [죽음] -> [영구 성장] -> ...
              |                          |
              v                          v
         Node 처치 자원 누적        더 오래 생존, 더 많은 자원
              |
              +-- 스테이지 클리어 시 프리미엄 자원 추가 획득
```

### 2.2 인게임 루프 (1회 런)

```
1. 스테이지 진입 (이전 배치 타워 유지, 새 스테이지면 초기화)
2. Node 웨이브 시작 (자동, 대기 없음)
3. 타워가 Node를 자동 공격
4. Node 처치 -> Bit(기본 자원) 획득 + 확률적으로 보물상자 드랍
5. Bit로 타워 추가 배치 / 인게임 업그레이드
6. 특정 웨이브 클리어 시 보물상자 보상 -> 3개 중 1개 선택 (런 내 버프)
7. 더 강한 웨이브 도착
8. 보스 웨이브 -> 보스 Node 처치 = 스테이지 클리어
8b. 기지 HP 0 -> 런 종료 (죽음)
9. 누적 Bit -> 영구 업그레이드 (Hub 화면)
10. 다시 1번으로
```

### 2.3 메타 루프 (스테이지 진행)

```
스테이지 1 반복 도전 -> 보스 처치로 클리어 -> Core(프리미엄 자원) 획득
         |                                      |
         v                                      v
    Bit/킬 수 누적 -----> 조건 달성 시 -----> 다음 스테이지 자동 해금
    (클리어 불필수)          |
                            v
                 스테이지 2 도전 -> 보스 처치로 클리어 -> ...
                            |
                            v
                 스테이지 10 보스 처치 = 엔딩
```

**핵심 변경**: 스테이지 해금은 이전 스테이지 클리어가 필수가 아니다. Bit 획득량, 총 킬 수 등 **누적 조건**으로 해금된다. 클리어하지 못해도 반복 플레이로 자연스럽게 다음 스테이지가 열리므로, 특정 스테이지에서 막히는 좌절감을 방지한다.

### 2.4 승리/패배 조건

- **런 종료 (죽음)**: 기지 HP가 0 이하. 패널티 없음. 획득한 Bit는 모두 유지.
- **스테이지 클리어**: 마지막 웨이브의 **보스 Node를 처치**하면 스테이지 클리어. Core 보상 획득.
- **게임 클리어 (엔딩)**: 스테이지 10의 최종 보스 처치.

설계 의도: 보스 처치가 클리어 조건이므로, 보스에 대한 명확한 목표의식이 생긴다. "보스만 잡으면 된다"는 단순한 목표가 긴장감과 달성감을 극대화한다. 죽음에 패널티가 없으므로 부담 없이 재도전하며, "한 판만 더"를 유도하는 핵심 구조.

## 3. 성장 시스템

### 3.1 자원 종류

| 자원 | 이름 | 획득 방법 | 용도 | 영속성 |
|------|------|----------|------|--------|
| Bit | 기본 자원 | Node 처치 시 드롭 | 인게임 타워 배치/업그레이드, 런 종료 후 영구 업그레이드 | 영구 누적 |
| Core | 프리미엄 자원 | 스테이지 클리어 보상 | 강력한 영구 업그레이드, 신규 타워 해금 | 영구 누적 |
| Idle Bit | 방치 자원 | 오프라인/방치 시간 경과 | Bit와 동일하게 사용 | 접속 시 수령 |

설계 의도: Bit는 "플레이하면 항상 얻는" 기본 보상으로 죽음의 좌절감을 상쇄한다. Core는 "스테이지를 깨야만 얻는" 희귀 보상으로 클리어 동기를 부여한다.

### 3.2 스킬 트리 (Hub 영구 업그레이드)

런이 끝나면 Hub 화면에서 **스킬 트리**를 통해 영구 업그레이드를 구매한다. Nodebuster 스타일의 트리형 시스템으로, 화면 중앙에 시작 노드 1개가 있고, 이를 찍으면 사방으로 선택지가 뻗어나간다.

#### 트리 구조 원칙

- **시작**: 화면 중앙에 "Core Node" 1개. 이것을 찍으면 상하좌우 4방향으로 첫 번째 분기가 열림
- **분기**: 각 노드를 찍을 때마다 연결된 다음 노드들이 보임 (2~3갈래)
- **잠금**: 연결되지 않은 노드는 보이지 않음. 찍을수록 트리가 점점 펼쳐지는 탐험감
- **다중 자원**: 초반 노드는 Bit로 구매, 깊은 노드일수록 Core 필요
- **모든 노드 해금 가능**: 영구적 제한 없이 충분히 플레이하면 전부 찍을 수 있음 (타이밍 최적화가 핵심)

#### 트리 시각 레이아웃

```
                        [스폰율 증가]
                             |
                [Bit 획득+] ─ [공격속도+]
                     |            |
          [HP 회복] ─ [사거리+] ─ [크리티컬]
               |          |          |
    [Idle 수집기] ─ [★ Core Node ★] ─ [Cannon 해금]
               |          |          |
     [배속 해금] ─ [공격력+] ─ [기지 HP+]
                     |            |
              [Ice 해금] ─ [시작 Bit+]
                             |
                    [Lightning 해금]
                             |
                      [Laser 해금]
                             |
                      [Void 해금]
```

(실제 트리는 더 많은 노드와 분기를 가짐. 위는 핵심 구조 예시)

### 3.3 타워 배치/합성 시스템 (인게임) [확정]

> **총괄PD 확정 (2026-02-15)**: 풀 합성(드래그&드롭) 방식 채택. GDD 원안 그대로 진행.

런 중 타워를 획득하고 배치하는 시스템. 스킬 트리에서 해금한 타워만 사용 가능.

#### 타워 획득

- 웨이브 클리어 시 보상으로 랜덤 타워 1기 지급 (해금된 타워 중 랜덤)
- Bit로 타워 직접 구매 가능 (하단 상점)
- 획득한 타워는 하단 **타워 인벤토리**에 쌓임

#### 타워 배치

1. 하단 인벤토리에 있는 타워를 **드래그**하여 맵의 빈 격자 칸에 놓으면 배치
2. 배치된 타워는 다음 런에서도 **자동 유지** (죽어도 배치 유지)
3. 새로 획득한 타워만 하단 인벤토리에 표시되어 추가 배치 가능

#### 타워 합성 (레벨업)

- **같은 타워 위에 드래그**: 같은 종류의 타워를 이미 배치된 같은 타워 위에 올리면 합성
- 합성 시 레벨 +1 (최대 Lv5)
- 합성 이펙트: 번쩍이는 강화 연출 + 수치 상승 팝업

```
예시:
Arrow Lv1 (배치됨) + Arrow Lv1 (인벤토리에서 드래그) = Arrow Lv2
Arrow Lv2 (배치됨) + Arrow Lv1 (드래그) = Arrow Lv3
```

#### 타워 판매

- 배치된 타워를 길게 클릭 → 판매 (Bit 70% 환불)
- 또는 인벤토리 밖으로 드래그하여 버리기

설계 의도: 드래그 배치 + 합성 시스템은 (1) 직관적인 조작감, (2) 같은 타워를 모아서 합치는 수집+성장의 쾌감, (3) "이 타워를 여기에 쓸까 합성에 쓸까?"라는 전술적 선택을 제공한다. 배치 유지로 매 런마다 재배치하는 번거로움을 제거.

### 3.4 성장 체감 설계

| 시점 | 영구 공격력 배율 (예상) | 체감 |
|------|---------------------|------|
| 첫 플레이 | x1.0 | 기본. 30초~1분에 죽음 |
| 5회차 | x1.5 | 눈에 띄게 더 오래 생존 |
| 15회차 | x2.5 | 스테이지 1 클리어 가능 |
| 30회차 | x4.0 | 스테이지 3~4 진행 |
| 50회차 | x8.0 | 폭발적 성장 구간. 화면이 이펙트로 가득 |
| 80회차~ | x15.0+ | 후반 스테이지. 대규모 Node 학살 |

설계 의도: 지수적 성장 곡선으로 "방금 전보다 확실히 강해졌다"를 매번 느끼게 한다. 특히 x4.0~x8.0 구간(1~2시간차)이 가장 재미있는 구간으로 설계.

### 3.5 보물상자 시스템 (런 내 랜덤 요소) [신규]

> **총괄PD 확정 (2026-02-15)**: 기본은 순수 영구 성장. 추가로 보물상자 획득 시 3개 중 1개 선택지 제공 (런 내 랜덤 요소).

#### 3.5.1 보물상자 획득 조건

| 획득 방법 | 조건 | 빈도 |
|----------|------|------|
| 웨이브 클리어 보상 | 홀수 번째 웨이브 클리어 시 확정 드랍 (W1, W3, W5...) | 스테이지당 2~10회 |
| Node 처치 드랍 | Node 처치 시 3% 확률로 드랍 | 런 내 랜덤 |
| 보스 처치 보상 | 보스 Node 처치 시 확정 드랍 (희귀 등급 보장) | 스테이지당 1회 |

#### 3.5.2 보물상자 열기

보물상자를 획득하면 화면에 **3개의 선택지**가 표시된다. 플레이어는 1개를 선택하며, 나머지 2개는 사라진다.

- 게임이 **자동 일시정지**되고 선택 UI가 표시됨
- 선택 시간 제한 없음 (신중하게 고를 수 있음)
- 선택한 버프는 **해당 런에서만 유효** (런 종료 시 소멸)

#### 3.5.3 보물상자 선택지 풀

**일반 등급 (웨이브 클리어/Node 드랍)**

| 선택지 | 효과 (런 내) | 설계 의도 |
|--------|-------------|----------|
| 화력 강화 | 타워 데미지 +15% | 즉시 체감되는 화력 |

| 집중 사격 | 타워 사거리 +0.5 | 더 먼 거리에서 공격 |

| 골드 러시 | Bit 획득 +30% | 자원 폭발 |
| 재빠른 손 | 타워 배치 비용 -20% | 더 많은 타워 배치 |
| 결속의 힘 | 같은 종류 타워 2기 이상 인접 시 데미지 +20% | 배치 전략 유도 |
| 회복의 파동 | 웨이브 클리어 시 기지 HP 3 회복 | 안정적 생존 |

**희귀 등급 (보스 처치/저확률 드랍)**

| 선택지 | 효과 (런 내) | 설계 의도 |
|--------|-------------|----------|
| 광폭화 | 타워 데미지 +40%, 기지 HP 최대치 -30% | 리스크/리턴 |
| 보물 사냥꾼 | Node 처치 시 보물상자 드랍률 2배 | 더 많은 선택지 |
| 연쇄 폭발 | Node 처치 시 25% 확률로 주변 1.5 범위 AoE | 연쇄 쾌감 |
| 시간 왜곡 | 모든 Node 이동속도 -20% | 강력한 글로벌 감속 |
| 과충전 | 다음 합성 시 타워가 +2레벨 (1회) | 즉시 고레벨 타워 |
| 거인 학살자 | Heavy/Boss Node에 데미지 2배 | 보스 특화 |

#### 3.5.4 선택지 규칙

- 3개의 선택지는 **동일 등급 풀에서 중복 없이** 랜덤 추출
- 같은 런 내에서 이미 선택한 버프와 **동일한 버프 재등장 가능** (중첩)
- 중첩 시 효과 합산 (화력 강화 2회 선택 = +30%)
- 희귀 등급 선택지는 3개 중 최대 1개만 포함

#### 3.5.5 설계 의도

보물상자 시스템은 순수 영구 성장의 "예측 가능한 반복"에 런 내 변주를 추가한다.

- **결정적이지 않은 랜덤**: 어떤 선택지를 골라도 런이 크게 달라지지 않지만, "이번 런은 화력 특화"처럼 미세한 차별화를 제공
- **선택의 재미**: 3개 중 1개를 고르는 행위 자체가 도파민 트리거
- **런 내 한정**: 영구 성장을 침범하지 않음. 죽으면 버프도 사라져서 "다음 런에서 또 뭐가 나올까?" 기대감 유지
- **보스 처치 보상 연계**: 보스를 잡으면 희귀 보물상자가 확정이므로, 보스 도전 동기 강화

**해금 조건 설계 근거**: 해금 조건 수치는 해당 스테이지를 도전하기 전까지 자연스럽게 달성되는 수준으로 설정. 스테이지 N-1을 반복 플레이하면서 축적되는 Bit/킬 수가 조건을 충족하도록 밸런싱. 클리어하지 못해도 반복 런만으로 다음 스테이지가 열리는 구조.

### 4.2 웨이브 규칙

- **자동 시작**: 웨이브 간 대기 없음. 이전 웨이브 종료 후 3초 뒤 자동 시작.
- **밀도 증가**: 웨이브가 진행될수록 스폰 간격이 짧아지고 Node 수가 증가.
- **보스 웨이브**: 각 스테이지의 마지막 웨이브에 보스 Node가 등장. 일반 Node와 함께 스폰.
- **클리어 판정**: 마지막 웨이브의 **보스 Node를 처치**하면 스테이지 클리어. 일반 Node가 남아있어도 보스 처치 시 즉시 클리어. [확정]
- **죽음 판정**: 기지 HP가 0 이하가 되면 즉시 런 종료. 보스 웨이브 중이어도 죽으면 런 종료.
- **보물상자**: 특정 웨이브 클리어 시 또는 Node 처치 시 확률적으로 보물상자 드랍 (3.5절 참조).

설계 의도: 보스 처치가 명확한 목표가 되어 "보스까지 버텨야 한다"는 긴장감과 "보스를 잡았다!"는 달성감을 극대화한다. 대기 시간을 최소화하여 "빠른 죽음 -> 빠른 재시도" 루프를 유지. 3초 간격은 타워를 1~2기 추가 배치할 수 있는 최소한의 여유.

## 5. 타워 시스템

### 5.1 타워 공통 동작

1. 사거리 내 가장 가까운 Node를 자동 타겟팅

4. 영구 업그레이드 배율이 곱연산으로 적용됨

### 5.3 타워 해금 (스킬 트리)

타워 해금은 Hub 스킬 트리의 Core 노드를 찍어야 한다. 트리에서 해당 노드까지 도달하려면 전제 노드들을 먼저 찍어야 함.

```
Arrow Tower: 시작 시 기본 제공
Cannon Tower: 트리에서 크리티컬 해금 → Cannon 해금 노드 (Core 3)
Ice Tower: 트리에서 공속 Lv5 → Ice 해금 노드 (Core 5)
Lightning Tower: Cannon + Ice 해금 → Lightning 해금 노드 (Core 8)
Laser Tower: Lightning 해금 → Laser 해금 노드 (Core 12)
Void Tower: Laser 해금 → Void 해금 노드 (Core 18)
```

설계 의도: 타워 해금이 트리의 깊은 곳에 위치하여, 그곳까지 찍어가는 과정 자체가 성장. 경로 위에 있는 Bit 노드들도 자연스럽게 찍게 되어 전체적인 파워 상승.

## 6. 적(Node) 시스템

### 6.1 Node 공통 동작

1. 스폰 포인트에서 생성되어 정해진 경로를 따라 기지로 이동
2. HP가 0이 되면 파괴, Bit 드롭 (화면에 숫자 팝업)
3. 기지에 도달하면 기지 HP를 깎고 사라짐
4. 스테이지가 올라갈수록 HP/속도에 스케일링 배율 적용

## 8. 콤보 보너스 시스템

### 8.1 개요

콤보 보너스 시스템은 연속 처치를 통해 Soul(Bit) 획득량을 폭발적으로 증가시키는 즉각 피드백 메카닉이다. 플레이어가 타워를 집중 배치하여 Node를 연달아 처치할수록 더 많은 보상을 얻는 구조로, "빠르게 잡을수록 이득"이라는 공격적 플레이를 유도한다.

### 8.2 콤보 판정 규칙

| 항목 | 값 |
|------|-----|
| 콤보 윈도우 | 3초 (마지막 Node 처치 후 3초 이내에 다음 처치 발생 시 콤보 유지) |
| 콤보 리셋 조건 | 3초 이내 처치 없음 (런 시작 시에도 리셋) |
| 콤보 카운트 | 연속 처치 누적 수. 상한 없음. |

### 8.3 콤보 구간별 보너스

| 콤보 구간 | Soul 보너스 배율 | 설명 |
|----------|---------------|------|
| 1 ~ 4 콤보 | +0% | 기본 드롭량 |
| 5 ~ 9 콤보 | +50% | 5연속 처치 달성 시 활성 |
| 10 ~ 19 콤보 | +100% | 10연속 처치 달성 시 활성 |
| 20+ 콤보 | +200% | 20연속 처치 달성 시 활성 |

보너스 Soul은 처치 시 기본 Soul 드롭량(TreasureManager 보너스 포함 후)에 배율을 곱하여 추가 지급된다.

### 8.4 콤보 피드백 연출

| 콤보 단계 | 시각/청각 피드백 |
|----------|--------------|
| 5+ 콤보 | 화면 상단 중앙 "COMBO x{N}!" 텍스트 팝업 (노란색, 크기 x1.2) |
| 10 콤보 정확히 도달 | 카메라 셰이크 (강도 0.15, 지속 0.2초) |
| 20+ 콤보 | "MEGA COMBO!" 텍스트 팝업 (고정 문구, 노란색, 크기 x1.2) |

텍스트 팝업 위치: 화면 기준 가로 50%, 세로 85% 지점 (월드 좌표 변환).

### 8.5 콤보와 경제 연계

GDD 7.1절 Bit 경제의 "콤보 보너스" 수입원은 본 시스템과 동일하다.

| 시나리오 | 결과 |
|---------|------|
| Bit Node(기본 드롭 3) 5연속 처치 시 | 3 * 1.5 = 4.5 → 반올림 5 Soul 추가 지급 (총 드롭: 8) |
| Bit Node 10연속 처치 시 | 3 * 2.0 = 6 Soul 추가 지급 (총 드롭: 9) |
| Bit Node 20연속 처치 시 | 3 * 3.0 = 9 Soul 추가 지급 (총 드롭: 12) |

설계 의도: 콤보 보너스는 타워 배치가 최적화된 후반 런에서 자연스럽게 발생한다. 초반에는 콤보를 유지하기 어렵지만, 성장 후 타워가 촘촘하게 배치되면 콤보가 끊이지 않아 "내가 강해졌다"를 자원 폭발로 체감하게 한다.

### 8.2 Node 처치 이펙트 스케일링

| 상황 | 이펙트 |
|------|--------|
| 일반 처치 | 작은 파티클 + "+3" 텍스트 |
| 연속 처치 (콤보) | 파티클 크기 증가 + 콤보 카운터 표시 + "COMBO x5!" |
| 대량 동시 처치 (AoE) | 큰 폭발 이펙트 + 화면 흔들림(미세) + 합산 Bit 표시 |

| 스테이지 클리어 | 전체 화면 연출 + Core 등장 애니메이션 |

설계 의도: 성장할수록 이펙트가 화려해진다. 처음에는 작은 파티클이 뜨지만, 후반에는 화면이 이펙트로 가득 차는 경험. "내가 이렇게 강해졌구나"를 시각적으로 체감.

### 8.3 보상 타이밍 설계

```
0초: 게임 시작
3초: 첫 Node 등장
5초: 첫 Node 처치 -> "+3 Bit" (첫 보상)
15초: 타워 1기 추가 배치 가능 (첫 성장)
30초~1분: 첫 죽음 -> Hub 진입 -> "영구 업그레이드 가능!" (핵심 후크)
1분: 두 번째 런 시작. 영구 업그레이드 덕에 더 오래 생존 -> "효과가 있네!" (확인)
2분: 웨이브 3~4 도달. 새로운 Node 타입 등장 -> "이건 어떻게 대응하지?" (호기심)
5분: 여러 번의 런으로 안정적 성장. 스테이지 1 클리어 가능성 체감 -> "곧 깰 수 있겠다" (목표)
```

- **상단 바**: 현재 웨이브, 보유 Bit, 기지 HP (체력바)
- **게임 맵**: 이미 배치된 타워 + 이동하는 Node들. 빈 격자가 배치 가능 슬롯
- **하단 - 타워 인벤토리**: 현재 보유 중인 미배치 타워들. 드래그로 배치
- **[+구매]**: Bit로 타워 즉시 구매 (해금된 타워 중 선택)
- **배속 + 일시정지**: 우측 하단
- 기지 HP 낮을 때 화면 테두리 붉은색 경고

### 10.3 타워 배치/합성 UX

#### 신규 배치

1. 하단 인벤토리에서 타워 아이콘을 **드래그**
2. 맵의 빈 격자 위로 가져가면 초록색 하이라이트
3. 손을 떼면 즉시 배치 (확인 없음 - 속도 우선)

#### 합성 (레벨업)

1. 인벤토리에서 타워를 **드래그**
2. 맵에 이미 배치된 **같은 종류** 타워 위로 가져가면 노란색 하이라이트 + "LV UP!" 표시
3. 손을 떼면 합성 → 레벨 +1 + 강화 이펙트 + 수치 팝업

```
합성 예시:
[Arrow Lv1] 위에 [Arrow] 드래그 → [Arrow Lv2] + 번쩍 이펙트
[Arrow Lv2] 위에 [Arrow] 드래그 → [Arrow Lv3] + 더 큰 이펙트
```

#### 타워 판매

- 배치된 타워를 **길게 클릭** → 판매 확인 → Bit 70% 환불
- 또는 배치된 타워를 드래그하여 **인벤토리 밖(화면 아래)으로** 끌면 판매

#### 배치 유지 규칙 [확정]

> **총괄PD 확정 (2026-02-15)**: 같은 스테이지 재시도 시 타워 배치 유지. 새로운 스테이지(다른 맵)로 이동하면 배치 초기화.

- **같은 스테이지 재시도**: 런 종료(죽음) 후 같은 스테이지를 다시 시작하면 **이전에 배치한 타워가 그대로 유지**
- **다른 스테이지 이동**: 새로운 스테이지(다른 맵)로 진입하면 **배치 전체 초기화** (모든 타워가 인벤토리로 복귀)
- 새로 획득/구매한 타워만 인벤토리에 표시되어 추가 배치 가능
- 합성으로 레벨업한 타워도 같은 스테이지 내에서는 유지

설계 의도: 같은 스테이지를 반복 도전할 때는 매번 재배치하는 번거로움을 제거한다. 새 스테이지에서는 맵 구조가 다르므로 기존 배치가 무의미하며, 새 맵에 맞는 전략을 세우는 재미를 제공한다.

## 14. 성장 단계별 플레이어 경험 타임라인

### 12.1 Phase 1: 온보딩 + 첫 도파민 (0~2분)

**목표**: "Node가 오고, 타워가 잡고, 자원을 얻는다"를 직관적으로 체득

| 시간 | 이벤트 | 플레이어 경험 | 감정 |
|------|--------|-------------|------|
| 0:00 | 게임 첫 시작. Hub 스킵, 바로 스테이지 1 진입 | "바로 시작이네?" | 호기심 |
| 0:03 | Arrow Tower 1기 자동 배치됨. 첫 Node 등장 | "뭔가 온다" | 긴장 |
| 0:05 | 타워가 Node를 자동 공격. Node 파괴. "+3 Bit" | "오, 잡힌다!" | 만족 |
| 0:15 | Bit 축적. "타워를 배치하세요" 안내 | "하나 더 놓아볼까" | 능동적 참여 |
| 0:30 | 웨이브 2~3. Node 수 증가. 기지 HP 감소 시작 | "좀 힘든데?" | 긴장 |
| 0:45~1:00 | 첫 죽음. "RUN COMPLETE" 화면 | "아, 죽었네" | 약간의 아쉬움 |

### 12.2 Phase 2: 첫 성장 체감 (2~5분)

**목표**: "성장하면 더 강해지고, 더 강해지면 더 많이 번다"를 체감

| 시간 | 이벤트 | 플레이어 경험 |
|------|--------|-------------|

### B.1 초반 밸런스 (스테이지 1, 영구 업그레이드 0)

- 기지 HP: 20
- Arrow Tower Lv1 DPS: 10.0

- 웨이브 1 (5마리): 1기로는 1~2마리 놓침 -> 기지 HP 18~19

- 웨이브 3 (Quick Node 등장): 높은 확률로 사망
- **결론**: 초기 상태에서 웨이브 2~3에 죽는 것이 정상. 이것이 "성장해야 한다"는 핵심 동기.

### B.2 첫 클리어 밸런스 (스테이지 1, 약 15~20회 런 후)

- 예상 영구 업그레이드: 공격력 Lv3(+30%), 공속 Lv2(+10%), 체력 Lv2(+10 HP)
- Arrow DPS 보정: 10.0 * 1.3 * 1.1 = 14.3
- 기지 HP: 30
- 이 수준이면 웨이브 5까지 안정적 클리어 가능

### B.3 후반 스케일링 검증 (스테이지 7)

- Node HP 배율: x22.0 -> Bit Node HP: 440
- 영구 공격력 예상: +200% (x3.0)

- Bit Node 처치 시간: 1.5초 -> 문제 없음
- Boss Node HP: 300 * 22.0 = 6600 -> 다수 타워 조합 필요 -> 적절한 난이도

### B.4 경제 밸런스 검증
