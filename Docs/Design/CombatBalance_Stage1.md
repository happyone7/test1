### 2.2 발수 기반 정밀 계산

Arrow Lv1: 데미지 8, 공격 간격 0.8초

**vs Bit Node (HP 20, 속도 1.5)**
```
사거리 진입 -> 통과까지: 3.0 * 2 / 1.5 = 4.0초
4.0초 동안 발사 가능: floor(4.0 / 0.8) + 1 = 6발
6발 * 8 데미지 = 48 데미지 (HP 20 초과 -> 처치 성공)
실제 처치 시점: 3발째 (8*3=24 > 20), 소요 시간 1.6초
잔여 시간: 4.0 - 1.6 = 2.4초 (다음 타겟으로 전환 가능)
```

**vs Quick Node (HP 12, 속도 3.0)**
```
사거리 진입 -> 통과까지: 3.0 * 2 / 3.0 = 2.0초
2.0초 동안 발사 가능: floor(2.0 / 0.8) + 1 = 3발
3발 * 8 데미지 = 24 데미지 (HP 12 초과 -> 처치 성공)
실제 처치 시점: 2발째 (8*2=16 > 12), 소요 시간 0.8초
잔여 시간: 2.0 - 0.8 = 1.2초 (충분한 여유)
```

**vs Heavy Node (HP 80, 속도 0.8)**
```
사거리 진입 -> 통과까지: 3.0 * 2 / 0.8 = 7.5초
7.5초 동안 발사 가능: floor(7.5 / 0.8) + 1 = 10발
10발 * 8 데미지 = 80 데미지 (HP 80 = 정확히 처치)
결론: Arrow 1기로 Heavy 1마리를 겨우 처치 가능 (전담 시)
문제: Heavy를 상대하는 동안 다른 Node가 통과함
```

## 3. 웨이브별 전투 시뮬레이션

### 3.1 첫 런 (영구 업그레이드 0, Arrow 1기)

#### Wave 1: Bit Node x5, 간격 1.2초, 대기 2초

```
시간  이벤트                                    기지 HP
0.0   런 시작. Arrow 1기 배치됨                  20/20
2.0   W1 시작. Bit#1 스폰
3.2   Bit#2 스폰
3.6   Bit#1 처치 (Arrow 3발, 1.6초 전투)        20/20  +3 Bit
4.4   Bit#3 스폰
4.8   Bit#2 처치                                20/20  +3 Bit (누적 6)
5.6   Bit#4 스폰
6.0   Bit#3 처치                                20/20  +3 Bit (누적 9)
6.8   Bit#5 스폰
7.2   Bit#4 처치                                20/20  +3 Bit (누적 12)
8.4   Bit#5 처치                                20/20  +3 Bit (누적 15)

결과: Bit#1~5 전부 처치. 기지 HP 20/20. Bit 15 획득.
```

**분석**: Arrow 1기로 Bit Node를 1.6초에 처치하고, 스폰 간격이 1.2초이므로 처치 후 0.4초 대기가 발생. 타이밍이 빡빡하지만 5마리 모두 처치 가능. 스폰 간격 1.2초 > 전환+발사 시간이므로 이론적으로는 누적 없이 처리 가능.

단, 실제로는 타겟 전환 딜레이, 투사체 비행 시간 등으로 1~2마리가 통과할 수 있음. **현실적 예상: 4~5마리 처치, 0~1마리 통과.**

**W1 웨이브 보너스**: 15 Bit (10 + 1*5 = 15)
**W1 종료 시 누적 Bit**: 12~15 + 15 = **27~30 Bit**

#### Wave 2: Bit Node x8, 간격 1.0초, 대기 3초

```
Arrow 1기 DPS 10.0
Bit Node 처치 시간: 1.6초
스폰 간격: 1.0초

문제: 1.0초마다 스폰, 1.6초마다 처치 -> 0.6초씩 밀림
8마리 스폰 완료: 7.0초 (0~7초)
8마리 전부 처치 시간: 12.8초 (1.6 * 8)

사거리 통과 시간 (Bit): 4.0초
-> 1기로는 4.0초 동안 최대 3마리 처치 가능
-> 나머지 5마리 중 일부 통과

현실적 계산:
- Arrow가 사거리 내에서 처치하는 Node: 약 5~6마리
- 통과하는 Node: 2~3마리
- 기지 데미지: 2~3

추가 타워 배치 가능성:
- W1에서 27~30 Bit 획득
- Arrow 1기 = 30 Bit -> 겨우 배치 가능!
- W2 시작 전 Arrow 추가 배치 시: 2기로 8마리 대부분 처리 가능
```

**시나리오 A: 추가 배치 못 함 (Bit 부족 or 놓침)**
- 통과: 2~3마리 -> 기지 HP: 17~18
- W2 종료 Bit: 15~18 + 20 (보너스) = 35~38
- 누적: 62~68 Bit (Arrow 1기 추가 배치 가능)

**시나리오 B: Arrow 추가 배치 성공 (2기)**
- 통과: 0~1마리 -> 기지 HP: 19~20
- W2 종료 Bit: 24 + 20 = 44
- 누적: 71~74 Bit - 30 (배치비) = **41~44 Bit** (추가 배치 여력)

**W2 웨이브 보너스**: 20 Bit (10 + 2*5 = 20)

#### Wave 3: Bit x6 + Quick x3 (실제 SO: Bit x9만), 간격 0.8초, 대기 3초

**중요 발견**: 현재 Wave_03 SO에는 Bit Node 9마리만 설정되어 있고, Quick Node가 빠져 있다. GDD에서는 Bit x6 + Quick x3로 설계되어 있다. **SO 수정 필요.**

GDD 기준으로 시뮬레이션:

```
구성: Bit x6 (HP 20) + Quick x3 (HP 12)
스폰 간격: 0.8초
총 스폰 시간: 8 * 0.8 = 6.4초

Arrow 2기 가정:
- 각 타워가 별도 타겟 -> 동시 2마리 처리
- Bit 처치: 1.6초, Quick 처치: 0.8초
- Quick은 빠르므로 (속도 3.0, 사거리 통과 2.0초) 우선 대응 필요

현실적 예상 (Arrow 2기):
- Bit 6마리: 5마리 처치, 1마리 통과 가능
- Quick 3마리: 2마리 처치, 1마리 통과 가능
  (Quick은 속도가 2배지만 HP가 낮아 2발에 처치)
- 총 통과: 1~2마리 -> 기지 데미지 1~2
```

**Arrow 2기, W3 결과**:
- 기지 HP: 16~19 (시나리오에 따라)
- Bit 수입: ~30 (Bit x5*3 + Quick x2*4 = 23) + 25 (보너스) = ~55
- 누적 Bit: ~96~99 (영구 업그레이드 1개 + 타워 1기 가능)

**Arrow 1기만으로 W3 도전 시**:
- Quick Node 3마리 중 2~3마리 통과 (빠른 속도)
- Bit Node 6마리 중 3~4마리 통과
- 기지 데미지: 5~7
- **기지 HP 13~15로 위험 수준 진입**

#### Wave 4: Bit x10 + Quick x5 (현 SO: Bit x15만), 간격 0.7초, 대기 3초

**SO 수정 필요**: 현재 Bit x15이나 GDD 기준 Bit x10 + Quick x5.

```
총 15마리, 스폰 간격 0.7초

Arrow 3기 가정 (런 중 추가 배치):

**SO 수정 필요**: 현재 Bit x14이나 GDD 기준 Bit x8 + Quick x4 + Heavy x2.

```

## 4. 런 회차별 시뮬레이션

### 4.1 영구 업그레이드 진행 시뮬레이션

**전제**: 매 런 종료 후 Hub에서 가장 효율적인 업그레이드 구매

#### 1~2회차 (누적 Bit ~200)

| 구매 | 비용 | 잔여 Bit | 효과 |
|------|------|---------|------|
| 기지 HP Lv1 | 30 (현 SO baseCost) | 170 | HP 20 -> 25 |
| 공격력 Lv1 | 50 | 120 | DMG x1.1 (8 -> 8.8) |
| 공격력 Lv2 | 65 | 55 | DMG x1.2 (8 -> 9.6) |

**주의**: Skill_BaseHp의 현재 SO baseCost=30이지만, GDD 권장값은 40. 아래 시뮬은 GDD 기준(40)으로 계산.

| 구매 (GDD 기준) | 비용 | 잔여 Bit | 효과 |
|------|------|---------|------|
| 기지 HP Lv1 | 40 | 160 | HP 20 -> 25 |
| 공격력 Lv1 | 50 | 110 | DMG x1.1 |
| 공격력 Lv2 | 65 | 45 | DMG x1.2 |

**2~3회차 런 예상**:
- DPS 12.0이면 Bit Node 처치: 1.6초 -> 1.6초 (발수 동일, 3발 = 26.4 > 20)
- 실질적으로 Bit 과잉킬 발생, Quick도 2발에 처치 유지
- 기지 HP 25로 증가 -> W3~W4까지 안정적 도달
- **도달 웨이브: W3~W4, Bit 획득: 약 120~170**

#### 3~5회차 (누적 Bit ~500)

| 추가 구매 (GDD 기준) | 비용 | 효과 |

**Arrow Lv1 보정 DPS**: 8 * 1.3 * (1.25 * 1.05) = 8 * 1.3 * 1.3125 = **13.65 DPS** (+36.5%)

**5회차 런 예상**:
- Bit Node 처치: 여전히 3발 (24 > 20) 이지만 공속 향상으로 간격 0.76초
- Quick Node 처치: 2발 유지, 간격 0.76초 -> 더 여유
- 기지 HP 30으로 여유 확보
- **도달 웨이브: W4, Bit 획득: 약 150~200**

#### 6~10회차 (누적 Bit ~1,000)

| 추가 구매 | 비용 | 효과 |
|------|------|------|
| 공격력 Lv4~5 | 110+143 = 253 | DMG x1.5 |

**Arrow Lv1 보정 DPS**: 8 * 1.5 * (1.25 * 1.15) = 8 * 1.5 * 1.4375 = **17.25 DPS** (+72.5%)

**10회차 런 예상**:
- Bit Node 처치: 2발 (12*1 = 12... 아니, 데미지 = 8*1.5 = 12, 2발 = 24 > 20) **2발 처치!**

- Quick Node 처치: 2발 (24 > 12) = 1.38초, 사거리 통과 2.0초 -> 여유
- Heavy Node 처치: 80/17.25 = 4.6초, 사거리 통과 7.5초 -> 처치 가능!
- **도달 웨이브: W4~W5 안정적, Bit 획득: 약 200~275**
- W5 Heavy 2마리도 대응 가능해지기 시작

### 6.3 수정 근거

**Skill_AttackSpeed baseCost 40 -> 80**:
- 현재 40이면 공격력(50)보다 싸서, 최적 투자 순서가 "공속 먼저"가 됨

**Skill_BaseHp baseCost 30 -> 40**:
- 현재 30이면 가장 싸서 무조건 첫 구매 대상이 됨
- 40으로 올려도 여전히 2번째로 싸지만, "공격력(50)과 고민할 수 있는" 수준
- GDD 의도: 기지 HP vs 공격력 중 선택의 재미

**Wave_03~05 Node 구성**:
- 현재 모든 웨이브가 Bit Node만 사용 -> W3~5 난이도가 GDD보다 쉬움
- Quick Node 부재로 "빠른 적" 대응 경험이 빠져 있음
- Heavy Node 부재로 W5의 "견고한 적" 긴장감이 없음
- GDD 기준대로 수정하여 다양한 Node 경험 제공

| Bit 드롭 | 50 | 큰 보상 |

### 7.2 보스 처치 시뮬레이션

현재 Stage 1에는 보스 웨이브가 없지만, GDD 6.3절 Node 수치에 보스가 정의되어 있으므로 W5 이후 또는 W5에 보스가 추가될 경우를 시뮬레이션한다.

**20회차 (첫 클리어 수준) Arrow 3기, 영구 DMG x2.0, 공속 x1.30**

```
Arrow Lv1 보정 데미지: 8 * 2.0 = 16
유효 데미지 (방어력 3): 16 - 3 = 13
보정 공격속도: 1.25 * 1.30 = 1.625/초
단일 Arrow 유효 DPS: 13 * 1.625 = 21.1

3기 집중 시 DPS: 21.1 * 3 = 63.4
보스 처치 시간: 300 / 63.4 = 4.7초

보스 사거리 통과 시간: 3.0 * 2 / 0.5 = 12.0초
여유: 12.0 - 4.7 = 7.3초 -> 충분히 처치 가능
```

**10회차 Arrow 2기, 영구 DMG x1.73, 공속 x1.15**

```
보정 데미지: 8 * 1.73 = 13.8
유효 데미지: 13.8 - 3 = 10.8
보정 공속: 1.25 * 1.15 = 1.4375
단일 DPS: 10.8 * 1.4375 = 15.5
2기 DPS: 31.0
처치 시간: 300 / 31.0 = 9.7초
사거리 통과: 12.0초 -> 겨우 처치 가능 (여유 2.3초)
```

**결론**: 보스는 약 10~15회차부터 처치 가능해지며, 20회차에는 안정적으로 처치 가능. 보스 도달 자체가 W5까지 생존을 전제하므로, 보스 처치 가능 시점과 W5 도달 시점이 거의 일치한다.
