## 1. 아트 비전 요약

Soulspire는 **다크 판타지 세계관의 픽셀 아트 타워 디펜스**이다. 어둡고 깊은 톤의 판타지 세계에서, 마법 타워들이 밀려오는 유기체 몬스터(Node)를 파괴하는 시각적 쾌감을 극대화한다. 3/4 쿼터뷰 픽셀 아트로 탑다운 TD의 전략적 가독성과 디테일한 스프라이트 표현을 동시에 확보하며, 미니멀한 배경 위에서 타워와 몬스터, 그리고 화려한 마법 이펙트가 시각적 주인공이 되는 구성을 추구한다. 성장할수록 이펙트 밀도와 스케일이 증가하여 "내가 강해졌다"를 눈으로 체감할 수 있도록 설계한다.

## 2. 세계관 키워드

### 2.1 세계관 전환

기존 "사이버 네온 / 데이터 네트워크" 세계관에서 **판타지 세계관**으로 전면 전환한다.

| 기존 (사이버) | 신규 (다크 판타지) |
|-------------|-----------------|
| Node = 데이터 패킷 | Node = 마계/차원의 유기체 몬스터 |
| Tower = 방화벽/보안 시스템 | Tower = 마법 타워/마탑 |
| Soul = 데이터 자원 | Soul = 마나 결정/소울 파편 |
| Core Fragment = 프리미엄 데이터 | Core Fragment = 고대 룬석/정수 |
| 회로기판 맵 | 황폐한 마법진/고대 유적 |
| 네온 발광 | 마법 발광/룬 광채 |

### 2.2 핵심 세계관 키워드

- **다크 판타지**: 어둡지만 절망적이지 않은 톤. 신비롭고 위험한 세계.
- **마법 요새**: 플레이어는 마법 타워를 건설하여 차원의 균열에서 밀려오는 몬스터를 막는다.
- **고대 유적**: 맵은 고대 문명의 유적/마법진. 미니멀하지만 신비로운 분위기.
- **성장 = 마법 각성**: 스킬 트리는 마법 각성 트리. 업그레이드는 룬 각인.
- **파괴의 쾌감**: 몬스터를 마법으로 파괴하는 화려한 이펙트가 핵심 피드백.

### 2.3 톤 매트릭스

| 축 | 위치 | 설명 |
|----|------|------|
| 밝음 -- **어두움** | 70~80% 어두움 | 배경과 기본 톤은 어둡다. 밝은 것은 마법 효과뿐. |
| 가벼움 -- **무거움** | 60% 무거움 | 시리어스하지만, 게임플레이 자체는 빠르고 경쾌. |
| 사실적 -- **양식적** | 80% 양식적 | 픽셀 아트의 양식적 표현. 리얼리즘 배제. |
| 귀여움 -- **위엄** | 65% 위엄 | 타워는 위엄 있고, 몬스터는 위협적. 귀여움 배제. |

## 3. 아트 스타일

### 3.1 픽셀 아트 + 3/4 쿼터뷰

| 항목 | 확정값 |
|------|-------|
| 렌더링 스타일 | 픽셀 아트 (수동 도트 or AI 생성 + 수작업 보정) |
| 시점 | 3/4 쿼터뷰 (아이소메트릭 유사, 약간 위에서 비스듬히 내려다봄) |
| 톤 | 다크 판타지 (어두운 배경 + 밝은 마법 하이라이트) |
| 라이팅 느낌 | 주변광 없음. 마법 효과와 타워/몬스터 자체 발광만 존재. |
| 외곽선 | 1~2px 검정 외곽선 (스프라이트 가독성 확보) |
| 안티앨리어싱 | 수동 AA (제한된 팔레트 내 중간색 사용) |

### 3.2 시점 상세: 3/4 쿼터뷰

```
카메라 앵글: 위에서 약 30~45도 각도로 내려다봄
                ╱╲
               ╱  ╲   <- 타워 윗면 일부 보임
              │ ■■ │  <- 타워 정면 보임
              │ ■■ │
              └────┘  <- 그림자/베이스 보임

특징:
- 오브젝트의 정면(앞면)과 윗면이 동시에 보임
- 높이감 표현 가능 (타워의 높이, 몬스터 크기)
- 탑다운보다 입체감 있지만, 사이드뷰보다 전략적 가독성 높음
```

### 4.2 팔레트 운용 원칙

1. **암부가 기본, 명부는 마법**: 화면의 80%는 어두운 톤. 밝은 색은 마법 효과, 피드백, UI 강조에만 사용.
2. **채도 제한**: 배경/환경은 채도 낮음(회색 계열). 마법/이펙트만 높은 채도.
3. **Floor별 색온도 변화**: 초반 Floor는 차가운 톤(청보라), 후반은 따뜻한 톤(적갈색, 금색).
4. **타워별 즉시 식별**: 타워 시그니처 컬러만으로 종류 구분 가능해야 함.
5. **몬스터 가독성**: 어두운 배경 위에서 몬스터 실루엣이 즉시 보여야 함. 몬스터는 배경보다 약간 밝은 톤.

### 4.3 Floor별 환경 색조

| Floor | 이름 | 환경 색조 | 분위기 |
|---------|------|----------|-------|
| 1 | Data Stream -> **잊혀진 초원** | 차가운 청록 (`#102020`) | 황혼의 초원. 첫 마물 출현. |
| 2 | Memory Block -> **무너진 성채** | 청회색 (`#181828`) | 버려진 성벽. 돌과 이끼. |
| 3 | Cache Layer -> **수정 동굴** | 보라 (`#1A1030`) | 수정이 빛나는 지하. |
| 4 | Pipeline -> **갈라진 협곡** | 주황갈색 (`#201810`) | 마른 협곡. 용암 흔적. |
| 5 | Processor Core -> **마탑 내부** | 붉은 보라 (`#201020`) | 마력이 소용돌이치는 탑 내부. |
| 6 | Bus Network -> **영혼의 숲** | 짙은 녹색 (`#101A10`) | 안개 낀 고사목 숲. |
| 7 | Kernel Space -> **심연의 사원** | 짙은 보라 (`#140A20`) | 고대 신전. 제단과 석상. |
| 8 | Overflow Zone -> **화산 지대** | 붉은 검정 (`#200808`) | 용암과 화산재. |
| 9 | Root Access -> **왕좌의 방** | 금색+검정 (`#1A1408`) | 타락한 왕좌. 금빛 룬. |
| 10 | Kernel Panic -> **차원의 균열** | 전 색조 혼합 | 모든 테마가 뒤섞인 카오스. |

제작 수량: 6종 x 최소 3단계(Lv1, Lv3, Lv5) = 18 스프라이트. Lv2/Lv4는 런타임 셰이더로 보간 가능.

### 5.3 맵/배경 - 미니멀/추상

#### 설계 철학

배경은 **최소한의 시각 정보만 제공**하여 타워, 몬스터, 이펙트에 시선을 집중시킨다.

```
레이어 구조 (뒤 -> 앞):
L0: 단색 배경 (Floor별 색조, 스크롤 없음)
L1: 미니멀 패턴 타일맵 (경로/배치칸 구분용, 낮은 채도)
L2: 장식 오브젝트 (최소한, Floor 분위기 전달용)
L3: 경로 하이라이트 (몬스터 이동 경로를 미묘하게 밝게)
L4: 타워/몬스터/이펙트 (주인공 레이어)
L5: UI 오버레이
```

#### 타일맵 설계

| 항목 | 값 |
|------|-----|
| 타일 크기 | 32x32 px (1 Unity 유닛 = 32px, PPU 32로 설정) |
| 타일 종류 | 경로 타일, 배치 가능 타일, 벽/장애물 타일, 빈 공간 |
| 팔레트 | Floor별 3~4색 그래디언트 (암->중->명) |
| 경로 표시 | 배치 타일보다 2~3톤 밝게. 미세한 룬 문양 패턴. |
| 배치 가능 표시 | 미세한 격자 점선 (평소 거의 안 보임, 타워 드래그 시 하이라이트) |

#### 장식 오브젝트 (Floor별)

각 Floor에 2~4종의 소형 장식을 배치하여 분위기를 전달. 게임플레이에 영향 없음.

- 잊혀진 초원: 마른 풀, 부서진 석비
- 무너진 성채: 금이 간 벽돌, 녹슨 창살
- 수정 동굴: 보라빛 수정 클러스터
- 갈라진 협곡: 갈라진 바위, 마른 나뭇가지
- 마탑 내부: 촛불 잔해, 부서진 책
- 영혼의 숲: 고사목 그루터기, 이끼
- 심연의 사원: 석상 파편, 제단 잔해
- 화산 지대: 용암 균열(발광), 화산재
- 왕좌의 방: 금빛 타일 조각, 깨진 왕관
- 차원의 균열: 공간 왜곡 이펙트, 혼합 파편

### 5.4 UI - 판타지 장식 프레임

#### UI 디자인 원칙

1. **정보 우선**: 모든 수치, 상태 정보는 즉시 읽힘
2. **판타지 프레임**: 패널 테두리에 석재/금속/룬 문양 장식
3. **낡은 양피지 톤**: 텍스트 배경에 약간의 텍스처 느낌 (그래픽 에셋으로 구현)
4. **최소 컬러**: UI는 기본적으로 무채색+금색. 기능별 색상은 아이콘/강조에만.

#### Sanctum 스킬 트리 UI 특수 처리

- 트리 배경: 양피지/마법서 펼친 느낌. 어두운 갈색 톤.
- 노드: 원형 룬 문양. 찍으면 발광.

### 5.5 VFX / 이펙트 - 화려하게

#### 이펙트 설계 원칙

1. **마법 이펙트 = 시각적 주인공**: 이펙트는 게임에서 가장 밝고 화려한 요소.
2. **타워 시그니처 컬러 준수**: 각 이펙트는 발사한 타워의 컬러를 따름.
3. **스케일링**: 성장할수록 이펙트 크기, 밀도, 지속시간 증가.
4. **계층별 차등**: 마이크로(작고 빠름) ~ 특대(화면 전체) 5단계.

#### 이펙트 목록

```

  ┌──┐             ╱──╲
  │  │           ╱ 윗면 ╲
  │정면│          │  정면  │

  └──┘          └──────┘
```

## 8. 기술적 고려사항

### 8.1 Unity URP 2D 설정

| 항목 | 설정 |
|------|------|
| Render Pipeline | URP 2D Renderer |
| 2D Lights | Point Light 2D (타워 마법 발광), Global Light 2D (환경) |
| Bloom | Post-processing Bloom으로 마법 발광 강조 |
| 스프라이트 필터링 | Point (No Filter) - 픽셀 아트 선명도 유지 |
| 압축 | None 또는 낮은 압축 (픽셀 깨짐 방지) |
| Max Size | 스프라이트별 적정 크기 (과도한 다운스케일 금지) |

### 8.2 2D Light 활용 계획

다크 판타지 톤에서 2D Light는 핵심적인 분위기 도구.

| 라이트 유형 | 용도 | 강도 |
|------------|------|------|
| Global Light 2D | 환경 전체 조명. 매우 어둡게 (0.2~0.4). | Floor별 조절 |
| Point Light 2D (타워) | 각 타워 주변 발광. 타워 시그니처 컬러. | 배치 시 자동 |
| Point Light 2D (이펙트) | 폭발/마법 순간 발광. 짧은 지속. | 이펙트 연동 |
| Spot Light 2D (경로) | 몬스터 경로 미묘한 하이라이트. | 낮은 강도 |

### 8.3 아틀라스 전략

| 아틀라스 | 내용 | 예상 크기 |
|---------|------|---------|
| `Atlas_Tower` | 타워 6종 x 3~5레벨 = 18~30 스프라이트 | 1024x1024 |
| `Atlas_Monster` | 몬스터 9종 x (이동4+피격2+파괴4) = ~90 스프라이트 | 2048x1024 |
| `Atlas_UI` | 프레임, 버튼, 아이콘, 바 등 | 1024x1024 |
| `Atlas_FX` | 이펙트 시트, 파티클 텍스처 | 1024x512 |
| `Atlas_Tile` | Floor 타일 (Floor별 분리 가능) | 512x512 x10 |

### 8.4 스프라이트 정렬 (Sorting)

3/4 쿼터뷰에서는 Y축 정렬이 필수.

| Sorting Layer | 순서 | 내용 |
|--------------|------|------|

| Tilemap | 10 | 타일맵 레이어 |
| Decoration | 20 | 장식 오브젝트 |
| Shadows | 25 | 타워/몬스터 그림자 |
| Units | 30 | 몬스터 (Y축 기준 동적 정렬) |
| Towers | 35 | 타워 (Y축 기준 동적 정렬) |
| Projectiles | 40 | 투사체 |
| Effects | 50 | 이펙트/파티클 |
| UI_World | 60 | 월드 스페이스 UI (데미지 팝업 등) |

### 8.5 성능 예산

### 10.3 스타일 락 프로세스 (재실행 필요)

기존 사이버 테마의 스타일 락이 무효화되었으므로, 다크 판타지 테마로 재실행한다.

3. **총괄PD 승인**: 스타일 락 결과를 PD에게 제시하여 최종 승인
