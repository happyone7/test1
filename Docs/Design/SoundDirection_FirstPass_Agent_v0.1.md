## 1. 사운드 디렉션 요약

### 1.1 톤 키워드

| 키워드 | 설명 |
|--------|------|
| **Dark Orchestral Minimal** | 풀 오케스트라가 아닌, 소편성 앙상블(현악 4~6중주 + 저역 목관/금관 1~2). 무겁되 뭉개지지 않음. |
| **Arcane Resonance** | 마법 이펙트음은 금속성 종/공명(resonant bell, singing bowl) 계열. 전자음 배제. |
| **Impact Clarity** | 타격/배치/합성 등 플레이어 액션 SFX는 어택이 뚜렷하고 짧음(0.5초 이하). 잔향은 짧게. |
| **Tension Gradient** | BGM이 전투 강도에 따라 레이어가 쌓이며 긴장감을 제어. 갑자기 시끄러워지지 않음. |
| **Earned Reward** | 성장/보상 사운드는 화려하되 과잉 않음. "여운이 남는 한 방" 지향. |

### 1.2 금지 요소

| 금지 | 이유 |
|------|------|
| 일렉트로닉/신스Incursion/네온 사운드 | 기존 사이버 테마 완전 폐기. 혼동 방지. |
| 8bit 칩튠 BGM | 픽셀아트이지만 톤은 다크 판타지. 칩튠은 가벼움. |
| 보컬/코러스 BGM | 인디 규모 상 품질 제어 어렵고 주의를 과도하게 끎. |
| 과도한 리버브/딜레이 | 타격 피드백 명확성 저하. 잔향은 0.3~0.8초 이내. |
| 점프스케어형 급격한 음량 변화 | 쾌적한 플레이 환경. 보스 등장도 볼륨 스파이크가 아닌 텍스처 변화로 표현. |

## 2. 레퍼런스 축

### 2.1 Darkest Dungeon (Red Hook Studios)

- **채택 이유**: 다크 판타지 톤의 교과서. 어둡고 무거운 현악+타악 조합, 긴장감 조절이 탁월.
- **참고 포인트**: 던전 탐색 BGM의 레이어 전환(평시->전투), 크리티컬 히트/스트레스 SFX의 임팩트 설계.
- **Soulspire 적용**: BGM 레이어 전환 구조, 보스 등장 시 음색 전환 기법.

### 2.2 Loop Hero (Four Quarters)

- **채택 이유**: 픽셀아트 + 다크 판타지 + 루프 게임플레이. 사운드가 "반복을 지루하지 않게" 만드는 설계.
- **참고 포인트**: 짧은 BGM 루프에서 변주를 주는 방법, 몬스터 처치의 건조하면서도 만족스러운 SFX.
- **Soulspire 적용**: Incursion 반복 구간 BGM 변주 전략, 처치음의 건조한 임팩트.

### 2.3 Kingdom: Two Crowns (Raw Fury)

- **채택 이유**: 픽셀아트 + 어두운 톤 + 환경 사운드스케이프 결합. 미니멀한 화면에서 소리가 공간감을 만듦.
- **참고 포인트**: 환경 앰비언스의 미니멀한 사용, 밤/위험 상태에서의 사운드 텍스처 변화.
- **Soulspire 적용**: Floor별 앰비언스 바닥 레이어, HP 저하 시 텍스처 변화.

### 2.4 Vampire Survivors (poncle)

- **채택 이유**: 저가 Steam 타이틀의 벤치마크. 이펙트 밀도가 높은 상황에서도 사운드가 뭉개지지 않는 믹싱.
- **참고 포인트**: 동시 다수 SFX 재생 시 우선순위 관리, 레벨업/진화 시 "보상감" SFX.
- **Soulspire 적용**: 다수 타워 동시 발사 시 폴리포니 제한 & 우선순위 설계, 합성/스킬 해금 보상음.

### 2.5 Slay the Spire (Mega Crit Games)

- **채택 이유**: "Spire" 네이밍 일관성 + 카드/전략 게임에서 턴마다 쌓이는 사운드 레이어 설계.
- **참고 포인트**: 카드 사용음의 짧고 명확한 설계, 보스전 BGM 전환, UI 인터랙션 SFX.
- **Soulspire 적용**: 타워 배치/합성 시 촉감 있는 짧은 SFX, 보스전 BGM 전환.

## 3. MVP 이벤트 사운드 매핑표

### 범례

- **우선순위**: P0 = MVP 필수, P1 = MVP 이후 즉시, P2 = 폴리싱
- **변형 수량**: 반복 청취 시 피로 방지를 위한 랜덤 변형 수

### 3.1 타워 공격 계열

| 이벤트 | 목적 | 사운드 캐릭터 | 길이 | 변형 수 | 우선순위 |
|--------|------|-------------|------|---------|----------|
| Arrow 발사 | 공격 행위 인지 | 짧은 현 뜯기(pluck) + 바람 가르기. 건조함. | 0.15~0.25초 | 3 | P0 |
| Arrow 적중 | 데미지 확인 | 둔탁한 작은 충격(thud). 살짝 금속성. | 0.1~0.2초 | 3 | P0 |
| Cannon 발사 | AoE 공격 시작 | 저역 "쿵"(deep thump) + 화염 훅. | 0.3~0.4초 | 2 | P1 |
| Cannon 폭발 | AoE 피해 확인 | 저역 폭발 + 파편 흩어짐. 가까울수록 크게. | 0.4~0.6초 | 2 | P1 |
| Ice 적중 | 감속 디버프 인지 | 유리/얼음 깨지는 소리(crackle). 고역 밝음. | 0.15~0.25초 | 2 | P1 |
| Lightning 체인 | 연쇄 타격 확인 | 전기 아크 "지직" x 체인 수. 빠르게 연달아. | 체인당 0.08초, 총 0.3~0.5초 | 2 | P1 |
| Laser 빔 지속 | 지속 데미지 인지 | 저~중역 웅웅거림(hum) 루프. 시작/종료 트랜지션 별도. | 루프 (2~3초 구간) | 1+트랜지션 2 | P2 |
| Void 장판 | 범위 데미지 인지 | 저역 맥동(pulse) 루프. 적이 범위 내일 때 강도 증가. | 루프 | 1 | P2 |

### 3.2 타워 배치/합성

| 이벤트 | 목적 | 사운드 캐릭터 | 길이 | 변형 수 | 우선순위 |
|--------|------|-------------|------|---------|----------|
| 타워 배치 | 전략적 결정 확인 | 돌 놓는 "톡" + 마법진 미세 공명. 무게감+신비감. | 0.3~0.5초 | 2 | P0 |
| 합성 (Lv2) | 성장 피드백 (소) | 배치음 + 밝은 종소리 1타. | 0.4~0.6초 | 1 | P0 |
| 합성 (Lv3) | 성장 피드백 (중) | 배치음 + 종소리 2연타 + 미세 화음. | 0.5~0.7초 | 1 | P1 |
| 합성 (Lv4/MAX) | 성장 피드백 (대) | 배치음 + 종소리 화음 + 저역 울림 + 미세 화면 흔들림 연동. | 0.8~1.2초 | 1 | P1 |

### 3.3 처치/보상

| 이벤트 | 목적 | 사운드 캐릭터 | 길이 | 변형 수 | 우선순위 |
|--------|------|-------------|------|---------|----------|
| 일반 Node 처치 | 처치 쾌감 | 짧은 파열(pop) + 약한 마나 흩어짐. 가볍고 빠름. | 0.15~0.25초 | 4 | P0 |
| Bit 드롭 | 자원 획득 인지 | 동전 같은 밝은 틴(ting). 경쾌. | 0.1초 | 3 | P0 |
| 크리티컬 히트 | 추가 피해 강조 | 일반 적중음 + 금속 충격 레이어(강). "쾅" 한 방. | 0.2~0.3초 | 2 | P1 |
| 보스 처치 | 최대 보상감 | 슬로우모션 연동: 깊은 폭발 -> 0.3초 정적 -> 승전 브라스 스팅(짧은 팡파르 2~3초). | 3~4초 | 1 | P1 |

### 3.4 Incursion/Floor 진행

| 이벤트 | 목적 | 사운드 캐릭터 | 길이 | 변형 수 | 우선순위 |
|--------|------|-------------|------|---------|----------|
| Incursion 시작 | 긴장감 알림 | 전쟁 나팔(horn) 한 음. 짧고 분명. | 0.5~0.8초 | 1 | P0 |
| Incursion 클리어 | 안도/보상 | 밝은 현악 상행 아르페지오 + 미세 종. | 0.8~1.2초 | 1 | P0 |
| 보스 등장 | 위협 인지 | 저역 드론(drone) 크레센도 + 금속 임팩트 1타. 슬로우모션과 동기화. | 1.5~2.0초 | 1 | P1 |
| Floor 클리어 팡파르 | 달성감 극대화 | 브라스+현악 팡파르(4~6초). 금빛 파티클 연출 동기화. Core Fragment 획득 시 추가 "반짝" 레이어. | 4~6초 | 1 | P1 |

### 3.5 기지/위험

| 이벤트 | 목적 | 사운드 캐릭터 | 길이 | 변형 수 | 우선순위 |
|--------|------|-------------|------|---------|----------|
| 기지 피격 | 피해 경고 | 둔중한 충격(deep impact) + 갈라지는 소리. | 0.3~0.5초 | 2 | P0 |
| HP < 30% (긴장) | 위기 인지 | BGM 레이어 전환(긴장 레이어 ON) + 심장박동 저역 펄스(0.8초 간격) 루프. | 루프 | 1 | P0 |
| HP < 10% (위급) | 즉각 대응 유도 | 심장박동 빨라짐(0.5초 간격) + 고역 경고 톤 추가. BGM 필터(하이컷) 적용. | 루프 | 1 | P1 |

## 4. BGM 전략

### 4.1 구간 정의

| 구간 | 분위기 | BPM 범위 | 키 성격 | 루프 길이 |
|------|--------|---------|---------|----------|
| **허브(Sanctum)** | 고요, 신비, 안전 | 60~75 | 단조 -> 장조 해결(밝은 여운) | 90~120초 |
| **인게임 - 평시** | 경계, 준비 | 90~110 | 단조, 낮은 음역 중심 | 60~90초 |
| **인게임 - 긴장** | 고조, 위기 | 110~125 | 단조, 타악 강화 | 위 루프 + 레이어 ON |
| **보스 구간** | 위압, 결전 | 120~135 | 단조, 저역 강화 | 45~60초 (보스전 전용 루프) |

### 4.2 레이어 구조 (인게임)

인게임 BGM은 **3레이어 구조**로 설계하여 런타임 전환:

```
Layer 0: 앰비언스 + 패드 (항상 재생, 볼륨 -18dB~-12dB)
         - Floor별 환경음 질감 (바람, 돌 부스러짐, 수정 공명 등)

Layer 1: 리듬 + 멜로디 (Incursion 시작 시 페이드인, 2~3초)
         - 현악 오스티나토 + 타악

Layer 2: 긴장 (HP<30% 또는 보스 Incursion 시 페이드인, 1~2초)
         - 저역 브라스 드론 + 빠른 타악 패턴 + 심장박동 펄스
```

### 4.3 크로스페이드 규칙

| 전환 | 페이드 방식 | 시간 |
|------|-----------|------|
| 허브 -> 인게임 | 허브 BGM 페이드아웃 -> 1초 무음 -> 인게임 Layer 0 페이드인 | 총 3초 |
| 평시 -> 긴장 (HP<30%) | Layer 2 페이드인 (Layer 0,1 유지) | 2초 |
| 긴장 -> 평시 (HP 회복) | Layer 2 페이드아웃 | 3초 (천천히 해소) |
| 일반 Incursion -> 보스 Incursion | Layer 1을 보스 전용 루프로 크로스페이드 + Layer 2 ON | 2초 |
| 보스 처치 -> Floor 클리어 | 전 레이어 급속 페이드아웃(0.3초) -> 팡파르 재생 | 0.3초 컷 |
| Incursion 클리어 -> 다음 Incursion 대기 | Layer 1 볼륨 -6dB 감소 (완전 끄지 않음) | 1초 |

### 4.4 Floor별 색조 (가정)

ArtDirection 문서의 Floor별 환경 색조에 대응하는 사운드 색조:

| Floor | 시각 테마 | 사운드 색조 (가정) |
|---------|----------|------------------|
| 1. 잊혀진 초원 | 차가운 청록 | 바람 + 가벼운 현악. 입문용으로 밝은 톤. |
| 2. 무너진 성채 | 청회색 | 돌/금속 질감 앰비언스. 약간 무거워짐. |
| 3. 수정 동굴 | 보라 | 결정 공명(크리스탈 톤). 에코 약간 증가. |
| 4. 갈라진 협곡 | 주황갈색 | 건조한 바람 + 저역 럼블. 따뜻한 톤. |
| 5. 마탑 내부 | 붉은 보라 | 마법 웅웅거림 + 촛불 크랙. 밀폐감. |

> Floor 6~10은 제작 단계(Phase 4)에서 추가 정의. 현재 MVP 범위는 Floor 1~2.

## 5. AI 제작 파이프라인 제안

### 5.1 도구 후보

| 도구 | 용도 | 상업 라이선스 | 비고 |
|------|------|-------------|------|
| **Udio** | BGM 생성 (프롬프트 기반) | 유료 플랜(Standard $10/월) 상업 이용 가능. 생성물 저작권은 사용자에게 귀속(이용약관 확인 필요). | 다크 오케스트럴 프롬프트에 강점. 루프 편집은 후처리 필요. |
| **Suno** | BGM 생성 (대안) | Pro 플랜($10/월) 상업 이용 가능. | Udio 대비 결과물 편차가 크나, 짧은 스팅/팡파르 생성에 유용. |
| **ElevenLabs SFX** | SFX 생성 (프롬프트 기반) | 유료 플랜 상업 이용 가능. | 타격음, 환경음 등 짧은 SFX에 적합. |
| **Freesound.org + 가공** | SFX 원본 소싱 | CC0/CC-BY 라이선스 개별 확인 필수. | AI 생성 결과가 불만족 시 대안. 수동 편집 필수. |

### 5.2 라이선스 체크포인트

| 단계 | 확인 사항 |
|------|---------|
| 도구 선택 시 | 이용약관에서 "commercial use" 명시 확인. 무료 플랜은 대부분 비상업적 용도만 허용. |
| 생성 시 | 생성물에 기존 곡 유사성(저작권 침해 리스크) 체크. AI 도구 내부 필터 활용. |
| 편집 후 | 최종 파일에 외부 샘플 사용 시 해당 샘플의 라이선스 별도 기록. |
| 출시 전 | Steam 업로드 전 전체 오디오 에셋 라이선스 목록 정리 + 총괄PD 서명. |

### 5.3 생성 -> Unity 반입 프로세스

```
1) 프롬프트 작성
   - BGM: "dark fantasy orchestral, minor key, strings + low brass,
           100 BPM, tower defense battle loop, no vocals"
   - SFX: "stone placement thud with subtle magic resonance, short,
           game sound effect, dark fantasy"

2) AI 생성 (Udio/Suno/ElevenLabs)
   - BGM: 2~4개 후보 생성 -> 가장 적합한 1개 선택
   - SFX: 5~10개 생성 -> 상위 2~3개 선택

3) 보정/편집 (DAW: Reaper 또는 Audacity)
   - BGM: 루프 포인트 편집, 레이어 분리(앰비언스/리듬/긴장),
          EQ/컴프레서, 라우드니스 맞춤
   - SFX: 불필요 구간 트림, 노이즈 제거, 어택/릴리즈 조정,
          변형 버전 생성(피치 +/-2~5%, 볼륨 +/-1~2dB)

4) 포맷 변환
   - 작업 파일: WAV 48kHz/24bit (보관)
   - Unity 납품: OGG Vorbis (BGM), WAV 44.1kHz/16bit (SFX)

5) Unity 반입
   - AudioClip 임포트 설정 적용 (섹션 6 참조)
   - 네이밍 규칙에 따라 폴더 배치
   - AudioMixer 버스 연결 + 볼륨 밸런싱

6) 인게임 테스트
   - 에디터 플레이 -> 이벤트별 사운드 트리거 확인
   - 동시 재생 상황 믹스 확인 (다수 타워 공격 시)
   - 라우드니스 미터 확인
```

## 6. Unity 납품 규격 초안

### 6.1 파일 포맷

| 카테고리 | 포맷 | 이유 |
|---------|------|------|
| BGM | OGG Vorbis (Quality 0.6~0.7) | 용량 절감 + 스트리밍 로드. WAV 대비 1/5~1/8 용량. |
| SFX (짧은) | WAV 44.1kHz / 16bit | 0.5초 이하 클립은 압축 오버헤드 > 용량 절감. 무압축이 지연 없음. |
| SFX (긴) | OGG Vorbis | 1초 이상 클립(환경음, 보스 연출 등)은 OGG로 용량 절감. |
| 앰비언스 루프 | OGG Vorbis | 스트리밍 로드. |

### 6.2 샘플레이트

| 용도 | 샘플레이트 | 비트뎁스 |
|------|-----------|---------|
| BGM | 44.1 kHz | - (OGG 압축) |
| SFX | 44.1 kHz | 16bit |
| 작업 원본 (보관) | 48 kHz | 24bit |

### 6.3 라우드니스 기준

| 버스 | 타겟 라우드니스 (LUFS) | 피크 리밋 (dBTP) |
|------|---------------------|-----------------|
| Master | -16 LUFS (가정) | -1.0 dBTP |
| BGM | -20 LUFS | -3.0 dBTP |
| SFX | -14 LUFS | -1.0 dBTP |
| UI | -18 LUFS | -3.0 dBTP |

> 가정: 마스터 라우드니스는 Steam/PC 게임 일반 기준. 총괄PD와 최종 확정 필요.

### 6.4 루프 규칙

| 카테고리 | 규칙 |
|---------|------|
| BGM 루프 | 파일 끝에서 시작으로 매끄럽게 이어져야 함. 마지막 1~2마디를 첫 마디에 겹치는 크로스페이드 구간 포함. Unity AudioSource `loop = true`. |
| 앰비언스 | 루프 바운더리에서 클릭/팝 없어야 함. 시작과 끝의 파형이 제로크로싱 근처. |
| SFX | 루프하지 않음 (one-shot). 예외: 심장박동, Laser/Void 지속음은 루프. |
| 팡파르/스팅 | 루프하지 않음. 재생 후 자동 정지. |

## 7. 리스크와 완화책

| # | 리스크 | 영향 | 완화책 |
|---|--------|------|--------|
| 1 | AI 생성 BGM 품질 편차 | 루프 연결 부자연스럽거나 장르 벗어남 | 후보 4개 이상 생성 -> 최선 선택 + DAW 수동 편집. 루프 포인트는 반드시 수동 설정. |
| 2 | AI 생성 SFX가 "게임용"답지 않음 | 어택이 약하거나 잔향 과다 | 트랜지언트 쉐이퍼(transient shaper)로 어택 강화 + EQ 하이패스로 저역 정리. |
| 3 | AI 도구 라이선스 변경 | 상업 이용 불가로 전환될 수 있음 | 결제 시점의 이용약관 스크린샷 보관. 대안 도구(Freesound + 수동 제작) 항시 확보. |
| 4 | 동시 SFX 재생 시 믹스 뭉개짐 | 타워 20개 동시 공격 시 사운드 과부하 | Unity AudioSource 최대 동시 재생 제한(폴리포니): 타워 SFX 8~12개, 총 SFX 24개. 우선순위: 배치/합성 > 기지피격 > 처치 > 발사. |
| 5 | Floor별 BGM 제작 부담 | 10Floor x 3레이어 = 30트랙 | MVP는 Floor 1 전용 BGM만 제작. Floor 2~4는 Layer 0(앰비언스)만 교체하고 Layer 1~2 공유. |
| 6 | 사운드 용량이 빌드 크기 압박 | Steam 업로드 시 불필요한 용량 증가 | OGG Quality 0.5~0.7 범위 내 최소 용량 설정. BGM 1곡당 2~3MB 이하 목표. 전체 오디오 50MB 이하. |

## 8. 요청자에게 필요한 추가 질문

### P0 (방향 결정에 필수 - 즉시 답변 필요)

1. **마스터 라우드니스 확정**: -16 LUFS를 기준으로 잡았는데, 다른 PC 게임 대비 볼륨 선호가 있는지?

3. **MVP 범위 BGM 수량**: Floor 1 전용 BGM(3레이어) + Sanctum BGM(1곡) + 보스 루프(1곡) + 팡파르(2곡) = 총 8트랙이 MVP 최소인데, 이 범위가 맞는지?

### P1 (제작 전 확인 필요)

4. **AI 도구 예산**: Udio/Suno 유료 플랜 구독 비용($10~20/월)을 집행해도 되는지?

6. **FTUE(첫 런) 전용 사운드**: 첫 런 진입 시 별도의 도입 사운드/나레이션이 필요한지, 아니면 일반 인게임 BGM으로 처리하는지?
7. **배속(TimeScale) 시 사운드 처리**: 일시정지 외에 2배속/3배속 기능이 있는지? 있다면 BGM 피치 변경 여부.

10. **타워별 고유 적중음 분리**: MVP에서는 Arrow만 별도, 나머지는 공유 적중음으로 가정. Phase 3에서 타워별 전용 적중음을 분리하는 것이 맞는지?
