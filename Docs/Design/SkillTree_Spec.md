## 1. 개요

Hub 화면의 핵심 콘텐츠인 **스킬트리**의 UI 상세 명세. UI팀장이 바로 구현할 수 있는 수준으로 작성한다.

### 1.1 스킬트리 목적

- 런 종료 후 획득한 Bit/Core로 **영구 업그레이드**를 구매하는 시스템
- 중앙 Core Node에서 시작하여 사방으로 뻗어나가는 트리 구조
- 찍을수록 트리가 펼쳐지는 **탐험감** 제공
- "다음에 뭘 찍을까?" 선택의 재미

### 1.2 화면 진입 조건

- 런 종료(죽음 또는 클리어) 후 Hub 화면 진입 시 자동 표시
- Hub 화면 = 스킬트리 화면 (Hub 전체가 스킬트리)

## 2. 전체 노드 목록 및 연결 관계

### 2.1 트리 구조도

아래는 전체 스킬트리의 노드 연결 관계이다. `-->` 는 전제 조건(선행 노드를 찍어야 다음 노드가 활성화)을 의미한다.

```
                         [N09: 스폰율 증가]
                              |
                   [N05: Bit 획득+] -- [N06: 공격속도+]
                        |                    |
             [N12: HP 회복] -- [N04: 사거리+] -- [N13: 크리티컬 해금]
                  |                 |                  |
       [N16: Idle 수집기] -- [N00: Core Node] -- [N14: Cannon 해금]
                  |                 |                  |
         [N17: 배속 해금] -- [N01: 공격력+] -- [N03: 기지 HP+]
                        |                    |
                   [N15: Ice 해금] -- [N07: 시작 Bit+]
                              |
                     [N10: Lightning 해금]
                              |
                       [N11: Laser 해금]
                              |
                       [N18: Void 해금]
```

|---------|----------|---------------------|----------|
| N00 | Core Node | 없음 (시작 노드) | N01, N03, N04, N06 |
| N01 | 공격력 강화 | N00 | N05, N15 |

## 3. 노드별 상세 명세

### 3.1 시작 노드

| 항목 | 값 |
|------|-----|
| ID | N00 |
| 이름 | Core Node |
| 설명 | 스킬트리의 시작점. 무료로 활성화 상태. |
| 비용 | 0 (자동 활성) |
| 최대 레벨 | 1 |
| 레벨당 효과 | 없음 (연결 노드 개방만) |

### 3.2 Bit 노드 (반복 구매형)

이 노드들은 Bit로 구매하며, 여러 레벨을 찍을 수 있다.

| ID | 이름 | 레벨당 효과 | 최대 레벨 | 기본 비용 | 비용 증가율 | 자원 |
|----|------|-----------|---------|----------|-----------|------|
| N01 | 공격력 강화 | 타워 데미지 +10% | 20 | 50 Bit | x1.3 | Bit |
| N03 | 기지 HP 강화 | 기지 HP +5 | 20 | 40 Bit | x1.25 | Bit |
| N04 | 사거리 강화 | 타워 사거리 +0.2 | 10 | 60 Bit | x1.3 | Bit |
| N05 | Bit 획득량 | Node 처치 Bit +15% | 15 | 100 Bit | x1.35 | Bit |
| N06 | 공격속도 강화 | 타워 공속 +5% | 15 | 80 Bit | x1.3 | Bit |
| N07 | 시작 Bit | 런 시작 Bit +30 | 10 | 120 Bit | x1.4 | Bit |
| N09 | 스폰율 증가 | Node 스폰 +10% | 10 | 90 Bit | x1.3 | Bit |
| N12 | HP 회복 | 기지 HP 초당 0.5 회복 | 5 | 200 Bit | x1.4 | Bit |

**비용 공식**: `기본비용 * (비용증가율 ^ (현재레벨))`, 소수점 반올림

|------|----------|----------|----------|
| 1 | 50 | 50 | +10% (x1.1) |
| 2 | 65 | 115 | +20% (x1.2) |
| 3 | 85 | 200 | +30% (x1.3) |
| 4 | 110 | 310 | +40% (x1.4) |
| 5 | 143 | 453 | +50% (x1.5) |
| 10 | 688 | 2,580 | +100% (x2.0) |
| 15 | 3,320 | 11,046 | +150% (x2.5) |
| 20 | 16,024 | 50,252 | +200% (x3.0) |

## 4. 트리 시각 레이아웃

### 4.1 그리드 좌표계

스킬트리는 **논리적 그리드** 위에 배치한다. 좌표 원점 (0, 0)은 Core Node의 위치이다.

- X축: 오른쪽이 양수, 왼쪽이 음수
- Y축: 위쪽이 양수, 아래쪽이 음수
- 그리드 1칸 = 노드 1개 크기 + 여백

### 4.2 노드 좌표 배치표

| 노드 ID | 이름 | 그리드 좌표 (X, Y) | 레이어 (Core로부터 거리) |
|---------|------|-------------------|----------------------|
| N00 | Core Node | (0, 0) | 0 |
| N01 | 공격력 강화 | (0, -1) | 1 |
| N03 | 기지 HP 강화 | (1, -1) | 1 |
| N04 | 사거리 강화 | (0, 1) | 1 |
| N06 | 공격속도 강화 | (1, 1) | 1 |
| N05 | Bit 획득량 | (-1, 1) | 2 |
| N07 | 시작 Bit | (2, -1) | 2 |
| N12 | HP 회복 | (-1, 1) 아래 -> (-2, 1) | 2 |
| N13 | 크리티컬 해금 | (2, 1) | 2 |
| N09 | 스폰율 증가 | (0, 2) | 3 |
| N15 | Ice 해금 | (1, -2) | 3 |
| N14 | Cannon 해금 | (3, 1) | 3 |

### 4.4 연결선 규격

| 연결 유형 | 스타일 | 색상 |
|----------|--------|------|
| 활성 연결 (양쪽 노드 모두 구매 완료) | 실선, 두께 3px, 발광 효과 | 밝은 청록 (#00FFCC) |
| 반활성 연결 (한쪽만 구매 완료) | 실선, 두께 2px | 흐린 청록 (#00FFCC, 40% 불투명) |
| 비활성 연결 (양쪽 모두 미구매) | 숨김 (보이지 않음) | - |

연결선은 노드 중앙에서 노드 중앙으로 직선 또는 L자 꺾임으로 그린다 (곡선 불필요, 프로토타입).

## 5. 노드 상태 정의

### 5.1 상태 목록

| 상태 | 조건 | 시각 표현 |
|------|------|----------|
| **숨김 (Hidden)** | 선행 노드가 하나도 구매되지 않음 | 노드 자체가 보이지 않음. 해당 위치에 빈 공간 |
| **잠김 (Locked)** | 선행 노드 중 최소 1개가 구매됨, 그러나 전제 조건 미충족 | 어둡고 흐린 아이콘. 자물쇠 표시. 클릭 시 "전제 조건" 안내 팝업 |
| **활성가능 (Available)** | 전제 조건 충족 + 자원 충분 | 밝은 테두리 반짝임 애니메이션. 비용 표시 |
| **활성가능 (자원 부족)** | 전제 조건 충족 + 자원 부족 | 밝은 테두리 (반짝임 없음). 비용 빨간색 표시 |
| **구매완료 (Purchased)** | 최소 1레벨 이상 구매됨 | 밝게 빛남. 현재 레벨 표시. 최대 레벨 미달 시 추가 구매 가능 |
| **최대레벨 (Maxed)** | 최대 레벨 도달 | 금색 테두리 + 별 이펙트. 더 이상 클릭 불가 |

### 5.2 상태 전이 다이어그램

```
숨김 --[선행 노드 1개 이상 구매]--> 잠김
잠김 --[전제 조건 모두 충족]-----> 활성가능
활성가능 --[구매 실행]------------> 구매완료
구매완료 --[추가 구매]------------> 구매완료 (레벨 증가)
구매완료 --[최대 레벨 도달]-------> 최대레벨
```

### 5.3 새 노드 등장 연출

선행 노드를 구매하여 새로운 노드가 `숨김 -> 잠김` 또는 `숨김 -> 활성가능`으로 전환될 때:

1. 해당 위치에 0.3초간 발광 파티클 (팡! 하고 나타나는 느낌)
2. 노드 아이콘이 스케일 0 -> 1로 0.2초간 팝업 (EaseOutBack)
3. 연결선이 선행 노드에서 새 노드 방향으로 0.3초간 그려지는 애니메이션

## 7. 구매 팝업 UI 상세

### 7.1 팝업 트리거

- 노드를 **클릭** 시 팝업 표시
- **숨김** 상태 노드는 클릭 불가
- **최대레벨** 노드 클릭 시 "최대 레벨 도달" 간단 메시지만 표시 (팝업 미표시)

### 7.2 구매 팝업 레이아웃 - Bit 노드 (반복 구매형)

```
+----------------------------------------------+
|                                                |
|   [아이콘]  공격력 강화                          |
|             Lv 3 -> Lv 4                       |
|                                                |
|   +---------+---------+                        |
|   | Before  | After   |                        |
|   +---------+---------+                        |
|   | x1.3    | x1.4    |                        |
|   | (현재)   | (+10%)  |                        |
|   +---------+---------+                        |
|                                                |
|   비용: 110 Bit                                |
|   보유: 250 Bit (구매 후 140 Bit)               |
|                                                |
|   [ 구매 ]          [ 닫기 ]                    |
|                                                |
+----------------------------------------------+
팝업 크기: 360x320 px
```

| 요소 | 설명 |
|------|------|
| 노드 이름 | 좌상단, 16pt Bold |
| 레벨 변화 | "Lv 3 -> Lv 4", 14pt. 화살표 부분 녹색 |
| Before/After 테이블 | 현재 효과 vs 구매 후 효과. After 열은 녹색 텍스트 |
| 비용 | 충분하면 흰색, 부족하면 빨간색 + [구매] 버튼 비활성 |
| 보유 자원 | 현재 보유량. 괄호 안에 구매 후 잔액 |
| [구매] 버튼 | 자원 충분 시 활성 (밝은 청록). 부족 시 비활성 (회색) |
| [닫기] 버튼 | 항상 활성. 또는 팝업 외부 클릭으로 닫기 |

### 7.3 구매 팝업 레이아웃 - Core 노드 (1회 구매형)

```
|                                                |
|   비용: 3 Core                                  |
|   보유: 5 Core (구매 후 2 Core)                  |
|                                                |
|   [ 해금 ]          [ 닫기 ]                    |
|                                                |
+----------------------------------------------+
```

| 차이점 | 설명 |
|--------|------|
| Before/After 대신 효과 설명 | 1회 구매형이므로 레벨 비교 불필요 |
| 타워 해금 시 타워 미리보기 | 타워 스프라이트 + 간단한 스탯 표시 |
| 버튼 텍스트 | "구매" 대신 "해금" |

### 7.4 구매 확인 플로우

```
[노드 클릭]
    |
    v
[팝업 표시] -- 자원 부족? --> [구매 버튼 비활성. 비용 빨간색]
    |                              |
    | 자원 충분                      | [닫기] 클릭
    v                              v
[구매 버튼 클릭]               [팝업 닫힘]
    |
    v
[구매 처리 (즉시, 추가 확인 없음)]
    |
    v
[구매 이펙트 재생]
    - 노드 발광 (0.5초)
    - Bit/Core 차감 애니메이션 (상단 자원 바에서 숫자 감소)
    - 레벨업 텍스트 팝업 "Lv 4!" (0.3초)
    |
    v
[팝업 내용 갱신 (다음 레벨 정보로)]
    - 최대 레벨 도달 시 팝업 자동 닫힘 + "MAX!" 텍스트
    |
    v
[새 노드 등장 체크]
    - 이번 구매로 새로 보이게 되는 노드가 있으면 등장 연출
```

**핵심**: 구매 후 팝업이 자동으로 닫히지 않고, 다음 레벨 정보로 갱신된다. 이를 통해 연속 구매가 가능하다 (빠른 레벨업 체감).

## 8. 줌/패닝 조작 방식

### 8.1 패닝 (화면 이동)

| 입력 | 동작 |
|------|------|
| 우클릭 드래그 | 트리 화면 이동 (패닝) |
| 마우스 휠 클릭 드래그 | 트리 화면 이동 (패닝, 대안) |
| 키보드 WASD | 트리 화면 이동 (초당 500px 속도) |
| 키보드 화살표 | 트리 화면 이동 (초당 500px 속도) |

### 8.2 줌 (확대/축소)

| 입력 | 동작 |
|------|------|
| 마우스 스크롤 위 | 줌 인 (마우스 커서 위치 기준) |
| 마우스 스크롤 아래 | 줌 아웃 (마우스 커서 위치 기준) |
| 키보드 + / - | 줌 인 / 줌 아웃 (화면 중앙 기준) |

### 8.3 줌 제한

| 항목 | 값 |
|------|-----|
| 최소 줌 (가장 멀리) | 0.4x (전체 트리 한눈에 보임) |
| 최대 줌 (가장 가까이) | 1.5x (개별 노드 상세 확인) |
| 기본 줌 | 1.0x |
| 줌 단계 (스크롤 1틱) | 0.1x |
| 줌 애니메이션 | 0.15초, EaseOutQuad |

### 8.4 패닝 제한

| 항목 | 값 |
|------|-----|
| 패닝 범위 | 트리 전체 바운더리 + 여백 200px |
| 패닝 감쇠 | 관성 있음 (드래그 종료 후 0.3초간 감속) |

### 8.5 시작 위치

- Hub 진입 시 **카메라가 Core Node(0,0) 중앙**에 위치
- 줌 레벨은 기본값(1.0x)
- 이전에 마지막으로 구매한 노드가 있으면 해당 노드 근처로 포커스 (부드러운 이동, 0.5초)

### 8.6 노드 클릭과 패닝 구분

| 상황 | 판정 |
|------|------|
| 좌클릭 (드래그 없음) | 노드 클릭 -> 구매 팝업 |
| 좌클릭 + 드래그 5px 이상 | 패닝으로 판정 (팝업 미표시) |
| 우클릭 + 드래그 | 항상 패닝 |

## 12. 트리 렌더링 구현 가이드 (UI팀장용)

### 12.1 권장 구현 방식

| 방식 | 설명 | 권장 |
|------|------|------|
| **ScrollRect + RectTransform** | Unity UI Canvas 위에 대형 RectTransform. ScrollRect으로 패닝, Scale로 줌 | 프로토타입 권장 |
| World Space Canvas | 3D 공간에 Canvas 배치. Camera 이동으로 패닝/줌 | 확장성 좋으나 구현 복잡 |

### 12.2 프로토타입 구현 순서 (권장)

1. **노드 프리팹**: SkillNodeUI 프리팹 제작 (아이콘, 이름, 레벨, 비용 텍스트)
2. **트리 컨테이너**: 대형 RectTransform 안에 노드들을 그리드 좌표로 배치
3. **연결선**: UI.LineRenderer 또는 Image (Stretched) 로 노드 간 선 그리기
4. **줌/패닝**: ScrollRect 커스텀 또는 Input 핸들러로 Scale + Position 조절
5. **구매 팝업**: 별도 Canvas (Overlay)에 팝업 프리팹
6. **노드 상태 관리**: SkillTreeManager에서 상태 조회, UI에 반영

### 12.3 데이터 바인딩

- `SkillTreeManager.GetNodeState(nodeId)` -> 현재 상태 조회
- `SkillTreeManager.GetNodeLevel(nodeId)` -> 현재 레벨
- `SkillTreeManager.CanPurchase(nodeId)` -> 구매 가능 여부
- `SkillTreeManager.Purchase(nodeId)` -> 구매 실행
- `SkillTreeManager.GetCost(nodeId)` -> 현재 비용
- `SkillTreeManager.GetEffect(nodeId)` -> 현재/다음 효과 수치

> 위 API는 프로그래밍팀장과 합의 필요. 프로토타입에서는 직접 SO 데이터 참조도 가능.

## 14. 프로토타입 생략 가능 항목

프로토타입 단계에서 시간이 부족할 경우 생략하고, 이후 이터레이션에서 추가할 수 있는 항목.

| 항목 | 우선순위 | 생략 시 대안 |
|------|---------|-------------|
| 노드 등장 애니메이션 | P1 | 즉시 표시 (애니메이션 없음) |
| 줌 기능 | P1 | 고정 줌 (1.0x). 패닝만 지원 |
| 패닝 관성 | P2 | 관성 없이 즉시 정지 |
| 연결선 활성화 애니메이션 | P2 | 즉시 색상 전환 |
| Idle Bit 수령 UI | P2 | 별도 팝업 없이 자동 수령 |
| 잠김 노드 "해당 노드로 이동" | P2 | 안내 텍스트만 표시 |
| 구매 이펙트 (발광, 텍스트 팝업) | P1 | 즉시 레벨 갱신 |

### 14.1 최소 구현 범위 (MVP)

1. 노드 배치 및 연결선 표시
2. 노드 상태 표시 (숨김/잠김/활성가능/구매완료)
3. 클릭 시 구매 팝업 (Before/After)
4. 구매 실행 및 자원 차감
5. 패닝 (우클릭 드래그)
6. 상단 자원 바, 하단 네비게이션
