# PC Platform Design Reference

## Platform Promise
Design for control depth, long sessions, customization, and content ecosystems that reward mastery and player agency.

## Core Constraints and Opportunities
- Keyboard and mouse precision.
- Wide hardware variance.
- Modding and community expectations.
- Rich service integrations and storefront features.

## Session Model
- Typical loop target: 20-90 minutes.
- Support quick resume and deep progression equally.
- Preserve continuity across long-form play.

## Input and UX Rules
- Bindable controls and profile presets.
- Clear support for keyboard, mouse, and optional controller.
- UI scaling for high DPI, ultra-wide, and multi-monitor setups.

## Feature Expectations
- Robust graphics and control settings.
- Save management and cloud sync assumptions.
- Achievement and social overlay integration planning.
- Optional mod support strategy where relevant.

## Economy and Live Operations
- Premium and DLC model clarity for player trust.
- Seasonal content plans for retention.
- Transparent communication around monetization boundaries.

## Storefront Planning
- Page narrative: clear fantasy, loop, and differentiation.
- Trailer and screenshot strategy tied to key game pillars.
- Feature tags aligned with actual design depth.

## Deliverables
- Platform UX specification.
- Input mapping and remap matrix.
- Hardware tier quality targets.
- Storefront readiness checklist.
- Live content cadence assumptions.

## Practical Checklist
- Can UI remain readable from 1080p to ultra-wide formats?
- Are critical controls remappable without conflicts?
- Is the game value proposition clear in storefront assets?
